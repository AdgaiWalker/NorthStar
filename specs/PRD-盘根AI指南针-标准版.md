## 2. 双站架构

> ⚠️ **核心架构决策**：本产品采用双前端 + 双后端架构，国内站与全球站完全分离，共享内容数据，不共享用户数据。

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────┐
│                    国内（中国大陆）                      │
│                   必须 ICP 备案                        │
│                                                     │
│   xyzidea.cn 前端（React + Vite）                   │
│           ↓                                          │
│   cn-backend（Node.js + Hono → 迁移至 Golang）     │
│           ↓                                          │
│   PostgreSQL（国内）                                │
│   ├── cn 用户表（手机号Hash、昵称、密码bcrypt）    │
│   ├── cn 收藏记录                                   │
│   └── 内容表（工具推荐、攻略文章）                    │
└──────────────────────────┬──────────────────────────┘
                           │
                    内容同步（CDC）
                    只读同步，用户表不互通
                           │
┌──────────────────────────▼──────────────────────────┐
│                   全球站（香港/新加坡）                  │
│                                                     │
│   xyzidea.com 前端（React + Vite）                  │
│           ↓                                          │
│   com-backend（Node.js + Hono → 迁移至 Golang）    │
│           ↓                                          │
│   PostgreSQL（海外）                                │
│   ├── com 用户表（GitHub ID、邮箱Hash、密码bcrypt）│
│   └── com 收藏记录                                  │
└─────────────────────────────────────────────────────┘
```

### 2.2 站点定义

> **核心原则：两站功能完全一致，仅内容不同。**
> 全球站和国内站共享同一套产品功能（搜索、AI 搜索、方案生成、收藏、导出等），区别仅在于内容领域和数据合规要求。

| 属性 | xyzidea.cn（国内站） | xyzidea.com（全球站） |
| --- | --- | --- |
| 定位 | 校园生活指南（面向大学生） | AI 工具推荐（面向全球用户） |
| 内容领域 | 日常起居 / 成长提升 / 精明消费 | 影视创作 / 编程开发 / 通用办公 |
| 服务器 | 中国大陆 | 香港或新加坡 |
| ICP 备案 | 必须 | 不需要 |
| 登录方式 | 手机号 + 邮箱 | GitHub OAuth + 邮箱 |
| 用户数据 | 手机号Hash、昵称、收藏记录 | GitHub ID、邮箱Hash（SHA-256）、昵称、收藏记录 |
| 客户端 | Web（核心）+ App（独立开发） | Web（核心）+ App（独立开发） |
| 欧盟用户 | N/A（中国大陆用户） | **只读不注册** |
| 站间链接 | Header 提供全球站跳转链接 | Header 提供国内站跳转链接 |

#### 2.2.1 功能一致性原则

以下功能在两站完全一致（交互、视觉、逻辑相同）：

| 功能 | 两站一致性 |
| --- | --- |
| 首页结构 | Hero + AI 搜索 + 领域切换器 + 模块切换器 + 内容网格 |
| AI 智能搜索 | 智谱 GLM-4-Flash，相同的搜索框、结果面板、额度机制 |
| 普通搜索 | 关键词过滤，相同的搜索框和结果展示 |
| 领域切换器 | Header 下拉菜单，切换领域后内容自动过滤 |
| 模块切换器 | 热门推荐 / 专题系列，胶囊按钮切换 |
| 文章页布局 | 三栏布局（系列目录 + 正文 + TOC） |
| TopicCard | 全高封面图 + 渐变遮罩 + 标题 + 描述 + 文章数 |
| ArticleCard | 横排图文（左图右文），标题 + 摘要 + 统计 |
| 方案生成 | 选工具 → 生成方案 → 保存 → 导出 |
| 收藏/导出 | 相同的收藏机制和导出格式（md/txt/csv） |

#### 2.2.2 内容差异化

两站的差异仅在内容层面：

| 维度 | 国内站（校园） | 全球站（AI 工具） |
| --- | --- | --- |
| 内容类型 | 文章 + 专题（校园攻略） | 工具 + 文章 + 专题（AI 推荐） |
| 领域 | 日常起居 / 成长提升 / 精明消费 | 影视创作 / 编程开发 / 通用办公 |
| 最小内容单元 | Article（校园文章） | Tool + Article（工具 + 文章） |
| AI 搜索系统提示 | "校园生活顾问" | "AI 软件顾问" |
| 特色功能 | 无（功能与全球站一致） | 无（功能与国内站一致） |

### 2.3 数据隔离原则（最高优先级）

以下原则**绝对不能违背**：

1. **cn 用户数据绝不流向海外节点** — 手机号等个人信息只能存于国内 PostgreSQL
2. **com 站点禁止收集任何敏感个人信息** — 绝对禁止：手机号、微信号、QQ号、身份证、生物特征
3. **用户账号体系完全独立** — 两站用户互不相通，不做单点登录
4. **账号注销后 30 天内物理删除** — CDC 确保两边同步清理
5. **内容同步为单向只读** — cn 站写，com 站只读同步，不反向写入

### 2.4 核心理念

- **功能一致，内容不同** — 两站是同一产品的两个实例，共享所有功能，仅内容领域不同
- **靠 AI 解决问题，不是单纯学 AI** — 用户带着问题来，带着方案走
- **站间互通** — 用户可在两站之间自由跳转，发现更丰富的内容
- 国内合规优先，**数据红线绝不踩**
- 全球服务通过 com 站实现，EU 用户只读不注册

## 3. 产品定义

### 3.1 产品定位

> 一句话定义：面向黑河学院学生与泛 AI 学习者的"工具 × 内容 × 路径"聚合检索与方案生成平台，同时面向全球用户提供 AI 工具推荐服务。核心理念：不是学 AI，而是靠 AI 解决问题。

### 3.2 双站产品愿景

**统一产品愿景**

盘根·AI 指南针是一个"内容 × AI 搜索 × 方案生成"平台，以领域化内容组织为核心，AI 智能搜索为特色，方案沉淀为留存抓手。两站共享同一套产品能力和交互体验，仅内容领域不同：

- **xyzidea.cn（国内站）**：校园生活指南 — 日常起居 / 成长提升 / 精明消费
- **xyzidea.com（全球站）**：AI 工具推荐 — 影视创作 / 编程开发 / 通用办公

用户核心路径（两站一致）：

```
输入问题 → AI 搜索/普通搜索 → 浏览推荐内容 → 加入工具/文章 → 生成方案 → 保存/导出
```

### 3.3 北极星体验

#### 3.3.1 体验定义（cn 站）

用户输入一个真实问题，系统输出一条**高质量、可执行**的路径（工具 + 资料 + 步骤），并允许一键沉淀为个人资产。

输出结构：

| 组件 | 数量 | 说明 |
| --- | --- | --- |
| 工具推荐 | 1–3 个 | 匹配用户需求的工具集合 |
| 内容推荐 | 1–2 个 | 相关教程或文章 |
| 方案步骤 | 3–5 步 | 可执行的行动路径 |
| 一键沉淀 | - | 保存到「我的方案」，支持导出 |

#### 3.3.2 情绪目标

| 时间节点 | 目标情绪 |
| --- | --- |
| 30 秒内 | 从"焦虑/迷茫"到"确定/可执行" |
| 2 分钟内 | 形成"我已经始推进"的主观感受 |

### 3.4 核心价值主张

#### 对用户的价值

- **明确路径**：减少工具筛选与学习路径探索时间
- **可复用方案**：降低同类任务的重复成本
- **工作流思维**：形成跨工具组合能力

#### 对平台的价值

- 以工具与内容为**基础资产**
- 以方案沉淀形成**用户留存抓手**
- 以共建机制形成**长期内容供给**

#### 商业原则

- 收费目标以覆盖模型调用与运营成本为底线，不以高利润为导向
- 主要售卖"AI 使用次数"（次数包）；通过"投稿激励次数"驱动共建

## 4. 用户研究

### 4.1 用户画像

#### 画像 A：新手执行者

| 维度 | 描述 |
| --- | --- |
| 特征 | 对 AI 工具了解有限，缺少清晰路线 |
| 痛点 | 不知道从哪一步开始，不知道选哪些工具 |
| 需求 | 可执行步骤与最低门槛的工具组合 |
| 成功标准 | 一次提问获得可跟做的步骤，产生第一份可保存方案 |

#### 画像 B：进阶整合者

| 维度 | 描述 |
| --- | --- |
| 特征 | 使用过多款 AI 工具，缺少稳定工作流 |
| 痛点 | 工具切换频繁，流程不稳定，复用困难 |
| 需求 | 以目标为中心的工具链，以及可复用模板 |
| 成功标准 | 形成可重复执行的流程，多次复用导出方案 |

#### 画像 C：内容学习者

| 维度 | 描述 |
| --- | --- |
| 特征 | 以教程学习为主，偏好视频与实战文章 |
| 痛点 | 内容分散，缺少与工具选择的对应关系 |
| 需求 | 内容推荐可直达，内容与工具绑定 |
| 成功标准 | 阅读内容后能回到工具选择并生成方案 |

### 4.2 核心任务（JTBD）

| JTBD | 任务描述 |
| --- | --- |
| JTBD-1 | **从问题到工具**：当我遇到具体任务时，我要快速找到最匹配的工具集合，从而立即开始 |
| JTBD-2 | **从工具到步骤**：当我选定工具时，我要得到分步骤路径，从而在最短时间拿到结果 |

## 6. 信息架构

### 6.1 领域划分

| 领域标识 | 领域名称 |
| --- | --- |
| creative | 影视创作 |
| dev | 编程开发 |
| work | 通用办公 |
| life | 生活专区 |

### 6.2 页面层级

> **两站页面结构完全一致**：首页 → 文章页 → 方案页 → 用户后台，相同的导航和交互模式。

#### 统一内容结构（两站一致）

```
Domain（领域）          ← 最高级分类，通过 Header 切换器切换
  ├── Tool（工具）      ← 按领域归类（全球站有，国内站无）
  ├── Topic（专题/系列）← 文章容器，同主题多篇关联文章
  │   └── Article（文章）← 最小内容单元（Markdown）
  └── Article（独立文章）← 不属于任何专题的文章
```

| 层级 | 全球站 | 国内站 |
| --- | --- | --- |
| 第一级 | Domain（影视创作/编程开发/通用办公） | Domain（日常起居/成长提升/精明消费） |
| 第二级 | Topic（系列主题） | Topic（专题） |

## 7. 用户旅程

### 7.1 旅程 1：普通搜索（免费）

```
用户进入首页
    ↓
用户在搜索框输入关键词
    ↓
选择模式：普通搜索
    ↓
系统展示搜索结果（工具/内容分组）
    ↓
用户进入工具详情或阅读页
```

### 7.2 旅程 2：AI 搜索（消耗次数）

```
用户进入首页
    ↓
用户在搜索框输入问题
    ↓
选择模式：AI 搜索
    ↓
系统检查额度
    ↓
若额度为 0：进入 demo 演示模式（不扣次数）
    ↓
选择库：专业库（精选）/综合库（全量）
    ↓
系统输出：摘要 + 路径建议 + 推荐工具 + 推荐内容
    ↓
用户加入工具/生成方案/保存导出
```

### 7.3 旅程 3：选择工具并生成方案

```
用户点击工具卡片上「加入方案」
    ↓
系统把工具加入已选集合
    ↓
系统展示浮动 Dock
    ↓
用户点击 Dock「生成方案」
    ↓
系统进入方案生成页
    ↓
用户输入目标描述
    ↓
用户点击生成
    ↓
系统生成方案内容
    ↓
用户点击保存
    ↓
系统将方案写入「我的方案」
```

### 7.4 旅程 4：管理方案并导出

```
用户进入「我的」
    ↓
用户进入「我的方案」
    ↓
用户点击某个方案
    ↓
系统打开方案详情抽屉
    ↓
用户点击导出
    ↓
系统生成文件并触发下载
    ↓
用户完成下载
```

### 7.5 旅程 5：站间跳转

```
用户在任一站浏览内容
    ↓
用户点击 Header 站间链接（国内站→全球站 或 全球站→国内站）
    ↓
系统在新标签页打开对方站点
    ↓
用户在对方站点继续浏览/搜索/收藏
    ↓
（可选）用户在对方站点注册账号
```

## 19. 风险管理

| 风险 | 影响 | 对策 |
| --- | --- | --- |
