## 1. 产品定位

> 盘根·AI 指南针是一个熵减器——把 AI 时代的混乱信息，通过实践验证，转化为有序的、可用的知识。

**三句话核心**：

1. **一半生活，一半理想** — 校园站（生活面）服务日常，全球站（理想面）服务创新
2. **从实验到应用** — idea 在理想站被探索，经过实践验证后，沉淀为生活站里普通人能直接用的知识
3. **推动 AI 大众化** — 帮助人们正确认识和使用 AI，实现个人价值与社会价值的统一

**底层驱动**：Ferry 框架（熵减思维：规划→执行→复盘）指导内容方向与质量保证。

## 2. 双站架构

双前端 + 单后端，两站共享后端逻辑，通过站点配置区分行为。部署时可拆为两个实例，代码不变。

| 属性    | xyzidea.cn（校园站·生活面） | xyzidea.com（全球站·理想面） |
| ----- | ------------------- | -------------------- |
| 当前用户  | 本校学生和老师             | 线上创作者、程序员等创作类工作者     |
| 内容领域  | 日常起居 / 成长提升 / 精明消费  | 影视创作 / 编程开发 / 通用办公   |
| 登录方式  | 手机号 + 验证码           | 申请制 + 邀请制 + GitHub OAuth |
| 服务器   | 中国大陆（ICP 备案）        | 香港/新加坡               |
| 待验证假设 | 扩展到校园所在区域的所有人       | 扩展到全球想用 AI 解决问题的人    |

**数据隔离红线**：cn 用户数据绝不流向海外；com 站禁止收集中国敏感个人信息；两站账号体系完全独立。

**合规底线**：

- cn 站上线前须完成：隐私政策页、用户协议页、公安备案
- 两站 AI 生成内容（方案、搜索结果）均须标注"由 AI 生成"
- cn 站接入的短信服务商须为持牌合规方（阿里云/腾讯云 SMS 等）
- cn 站用户享有账号注销与数据删除权利，用户后台须提供入口

## 3. 用户画像与知识流

> 基于实际调查数据，非假设。

| 画像         | 来源   | 痛点                                             | 需求                      |
| ---------- | ---- | ---------------------------------------------- | ----------------------- |
| 本校学生       | 校园调查 | 不知道怎么用 AI 解决日常问题                               | 食堂、选课、省钱等场景的 AI 实用教程    |
| 本校老师       | 校园调查 | 想用 AI 辅助教学但不知从何入手                           | 备课、出题、科研等场景的 AI 实用教程   |
| 线上创作者      | 线上调研 | 有 idea 但缺好用工具及落地方法 | AI 音乐、AI 绘画、AI 视频的实验与案例 |
| 程序员/创作类工作者 | 线上调研 | 工具多但工作流不稳定                                     | 以目标为中心的工具链，可复用方案        |

**知识流**：用户不按站划分。同一个人的 idea 在理想站被探索，实践验证有用后，沉淀到生活站给普通人直接用。形成 **实验 → 验证 → 应用** 的转化闭环。

每个产品（FrontAI / FrontLife）覆盖三个平台，内容与后端一致，前端与交互各平台独立设计：

| 阶段 | 平台 | 说明 |
|---|---|---|

| 下一阶段 | 小程序 | 微信小程序，校园站优先 |
| 未来 | App | 原生 App（iOS / Android），功能从 Web 平移 |

架构约束：共享代码层（类型、API 契约、工具函数）必须为纯 TypeScript，不绑定任何前端框架，确保 Web / 小程序 / App 均可复用。

## 9. 内容安全：敏感词管控

### 设计原则

- **两层防护**：输入拦截（不发 AI 请求）+ 输出过滤（AI 响应降级）
- **词库共享**：敏感词库位于 `@ns/shared`，两站共用同一份词库
- **降级策略**：命中敏感词时不报错，而是降级为本地关键词搜索（fallback），保证用户体验

### 敏感词分类

| 分类 | 示例方向 | 说明 |
|---|---|---|
| 政治 | 依法律法规动态调整 | 需配合监管要求实时更新 |
| 色情 | 色情、裸体、成人视频等 | 覆盖直接描述与常见变体 |
| 暴力 | 杀人、自制武器、自残等 | 含自杀方法类 |
| 赌博 | 赌博、博彩、百家乐等 | 覆盖线上赌场术语 |
| 违禁品 | 代开发票、买卖枪支、迷药等 | 含金融欺诈类 |
| 辱骂 | 傻逼、操你等 | 覆盖常见脏话 |

### 拦截流程

```
用户输入 query
  → checkSensitiveWords(query) 命中？
    → 是：展示"搜索内容不符合规范"提示（frontlife）/ 降级为 fallback 结果（frontai）
    → 否：发送 AI 请求
      → AI 返回 summary
        → checkSensitiveWords(summary) 命中？
          → 是：降级为 fallback 关键词搜索（FallbackReason: sensitive_output）
          → 否：正常展示 AI 结果
```

### 技术实现

- **词库文件**：`packages/shared/src/sensitive.ts`
- **校验函数**：`checkSensitiveWords(text)` → `{ hit: boolean, words: string[] }`
- **FallbackReason 新增**：`sensitive_blocked`（输入拦截）、`sensitive_output`（输出过滤）
- **原型阶段**：客户端硬编码词库（~50 词），后续迁移至后端词库 + 增量更新

### 演进路线

| 阶段 | 词库管理 | 匹配算法 | 覆盖范围 |
|---|---|---|---|
| 原型（当前） | 客户端硬编码 | 字符串包含匹配 | 6 分类 ~50 词 |
| 后端搭建后 | 服务端词库 + 后台管理 | DFA 自动机 / 正则变体 | 持续扩充 |
| 正式上线 | 第三方内容安全 API | NLP 语义分析 | 全量覆盖 |
