## 二、架构设计差异

### 2.1 Identity 模型：Account/Credential/Profile 三表 vs 单 users 表

**架构设计要求**：拆分为 Account + Credential + CampusProfile + CompassProfile + LevelChangeLog 五个模型。

**代码现状**：单 `users` 表包含 username、email、passwordHash、githubId、wxOpenId、phone、nickname、avatar、school、role、trustLevel、site 等全部字段。

**影响**：
- 功能层面可正常运行，当前阶段无实际影响
- 未来多绑定方式（邮箱、手机、GitHub、微信）需 credential 表隔离
- 校园/全球 profile 分离需拆表

**建议**：当前暂不拆，列为 P7 技术债。

### 2.2 API 旧路径迁移

**架构设计要求**：旧 `/api/spaces`、`/api/articles`、`/api/posts` 等路径应迁移到 `/api/campus/*` 命名空间。

**代码现状**：
- 校园站前端已全部调用 `/api/campus/*` 新路径 ✅
- 后端 `src/routes/` 目录下仍存在 `auth.ts`（旧 `/api/auth/*`）和 `reports.ts`（旧 `/api/reports`）
- `src/data/postgres.ts` 作为早期数据层仍被 campus 模块引用

**建议**：`/api/auth/*` 保留到前端全部切换到 `/api/identity/*` 后删除。

### 2.3 Feature Flag 专属 UI

**架构设计要求**：后台有"功能开放控制台"，支持 Hidden → Locked → Degraded → Enabled 四态切换。

**代码现状**：
- 后端 `GET /api/platform/feature-flags?site=` 已实现 ✅
- Feature flag 数据存于 `site_configs` 表，key 前缀 `feature_flag_` ✅
- admin-console 无专属 Feature Flag 页面，只能通过"系统配置"页的通用 JSON 编辑器修改 ⚠️

### 2.4 学校报告工作台

**架构设计要求**：后台应有"学校报告工作台"模块。

**代码现状**：**完全不存在**。无表、无路由、无前端页面。

### 2.5 需求洞察工作台

**架构设计要求**：后台应有"需求洞察工作台"，对搜索缺口、高频需求进行聚类分析。

**代码现状**：
- 后端有 `GET /api/campus/search/gaps` 返回搜索缺口数据 ✅
- analytics 模块有指标聚合 ✅
- admin-console 无专属洞察页面，仅在"数据中心"展示原始事件列表 ⚠️

## 七、跨包一致性问题

### 7.1 前端 mock 遗留

- 校园站：mock 代码已清除 ✅
- 全球站：`AIService.ts` 通过 `API_ENDPOINTS.AI_CHAT` 调用 `/api/ai-gateway/chat`，已走后端 Gateway ✅

### 7.2 共享类型与实际 API 不对齐

- `@ns/shared` 定义了 `CompassCapabilityResponse` 类型，但实际 capability 返回字段更多（如 `canPost`、`canWriteArticle`）
- `api-types.ts` 中 `ArticleDetail.author` 有 `helpedCount` 但无 `level` 字段

### 7.3 校园站 store 拆分 vs 全球站 store 合并

- 校园站使用 4 个独立 store（`useUserStore`、`useSearchStore`、`useSpaceStore`、`useUIStore`）
- 全球站使用 1 个合并 store（`useAppStore`）
- 两者模式不同，无统一规范要求，但维护成本各异

## 八、优先级建议

按架构设计 §13 的优先级框架，以下是建议执行顺序：

### P3 — 功能闭环（当前应做）

1. **全球站法律页面**：创建路由和页面组件，调用 `/api/compliance/legal-documents`
2. **全球站密码重置入口**：后端已就绪，前端加忘记密码表单
3. **校园站搜索 Overlay**：实现全局搜索浮层组件

### P4 — 体验优化

5. **内容工作室 UI（全球站）**：利用已有 `MarkdownEditor.tsx` 和后端 Content CRUD，创建内容编辑页面
6. **信任等级前端展示**：在校园林"`我的`"页和文章作者信息中展示等级
7. **空间认领前端入口**：在探索页或空间页添加认领触发
8. **Moderation 任务去重**

### P5 — 上线准备

9. **生产级邮件 provider**：替换 `devEmailProvider` 为 SMTP/SendGrid
10. **后台合规处理页面**：注销审批、数据导出审批
11. **后台 Feature Flag 专属面板**
12. **需求洞察工作台**
13. **AI 成本看板**
14. **行为数据 90 天自动清理定时任务**

### P6+ — 远期

16. **学校报告工作台**
17. **Identity 模型拆分**
18. **支付 provider 接入**
