# M5 生成方案页规范

## 页面概述
- **路由**: `/solution/generate?toolIds={ids}` 或 `/solution/:id`
- **优先级**: Phase 2（核心搜索与方案生成）
- **依赖**: AI服务接口、工具数据
- **核心功能**: 基于用户选择的工具，AI生成完整的使用方案

## 功能模块

### 1. Header
- **返回按钮**
  - 图标：左箭头
  - 点击：返回搜索页或首页
  - 确认提示（如果方案未保存）："确定要离开吗？未保存的方案将丢失"
- **标题**
  - 生成中："正在生成方案..."
  - 生成完成："方案详情"
- **分享按钮**
  - 图标：分享
  - 仅在方案生成完成后显示
  - 点击：弹出分享菜单

### 2. 加载状态
- **显示条件**
  - AI正在生成方案时
- **内容**
  - 旋转加载动画（圆环或自定义图标）
  - 文字提示："AI正在为您生成专属方案..."
  - 进度提示（可选）："分析工具特性..." → "制定使用步骤..." → "优化方案细节..."
- **交互**
  - 生成完成后：Medium震动反馈
  - 淡出加载状态，淡入方案内容

### 3. 方案Header区域
- **方案标题**
  - AI自动生成（基于工具组合和目标）
  - 字号：2xl-3xl
  - 可编辑（点击后变为输入框）
- **方案元信息**
  - 基于X个工具
  - X个步骤
  - 预计耗时（AI估算）
  - 难度级别（简单/中等/困难）
- **标签**
  - 领域标签（creative/dev/work）
  - 自动生成的关键词标签

### 4. 使用的工具区域
- **标题**
  - "使用的工具"
- **工具列表**
  - 工具图标 + 名称
  - 简短描述
  - 在方案中的作用说明（AI生成）
  - 点击工具：跳转到工具详情页
- **样式**
  - 卡片列表
  - 可横向滚动（移动端）
  - 网格布局（桌面端）

### 5. 方案步骤区域
- **标题**
  - "实施步骤"或"操作指南"
- **步骤列表**
  - 每个步骤包含：
    - 步骤编号（1、2、3...）
    - 步骤标题（AI生成）
    - 详细说明（支持Markdown格式）
    - 涉及的工具（高亮显示）
    - 预计耗时
    - 提示/注意事项（可选）
    - 示例图片（可选）
- **交互**
  - 步骤可折叠/展开
  - 完成状态勾选（本地记录）
  - 步骤之间的连接线（视觉引导）
- **样式**
  - 时间轴布局（左侧编号，右侧内容）
  - 或卡片布局（独立卡片）

### 6. AI建议区域
- **标题**
  - "AI的建议"或"温馨提示"
- **内容**
  - AI生成的额外建议
  - 常见问题预警
  - 优化技巧
  - 相关资源推荐
- **样式**
  - 浅色背景卡片
  - 图标：灯泡或AI图标

### 7. 底部操作栏
- **保存按钮**
  - 主按钮样式
  - 文字："保存方案"
  - 点击：
    - 调用API保存方案到用户账号
    - Toast提示："已保存到我的方案"
    - Medium Haptic
    - 跳转到个人中心-我的方案页
- **复制链接按钮**
  - 次要按钮样式
  - 文字："复制分享链接"
  - 点击：
    - 生成分享链接
    - 复制到剪贴板
    - Toast提示："链接已复制"
    - Light Haptic
- **导出按钮（可选）**
  - 导出为Markdown
  - 导出为PDF
  - 导出为图片

### 8. 分享菜单（弹窗）
- **触发**
  - 点击Header分享按钮
  - 点击"复制链接"按钮
- **内容**
  - 复制文本（包含方案摘要）
  - 分享到微信
  - 分享到QQ
  - 生成分享海报
- **交互**
  - 点击选项：执行分享操作
  - 点击外部区域：关闭菜单

### 9. 编辑模式（可选）
- **触发条件**
  - 已保存的方案
  - 用户是方案创建者
- **功能**
  - 编辑方案标题
  - 编辑步骤内容
  - 添加/删除步骤
  - 重新排序步骤
- **操作**
  - "编辑"按钮（右上角）
  - 点击后进入编辑模式
  - "保存"/"取消"按钮

## 状态管理

```typescript
interface SolutionGenerateState {
  // 生成相关
  isGenerating: boolean;
  generationProgress: string; // "analyzing" | "planning" | "optimizing"
  
  // 方案数据
  solution: UserSolution | null;
  selectedTools: Tool[];
  
  // UI状态
  isEditMode: boolean;
  showShareMenu: boolean;
  completedSteps: Set<number>; // 用户勾选的完成步骤
  
  // 保存状态
  isSaved: boolean;
  isSaving: boolean;
}

interface UserSolution {
  id: string;
  title: string;
  targetGoal: string;
  toolIds: string[];
  steps: SolutionStep[];
  aiAdvice: string;
  estimatedTime: string;
  difficulty: 'easy' | 'medium' | 'hard';
  tags: string[];
  createdAt: string;
  updatedAt: string;
}

interface SolutionStep {
  stepNumber: number;
  title: string;
  content: string; // Markdown格式
  toolsUsed: string[]; // 工具ID数组
  estimatedTime: string;
  tips?: string;
}
```

## API接口

### 生成方案
```
POST /api/solutions/generate
Request: {
  toolIds: string[],
  userGoal?: string, // 可选的用户目标描述
  domain: Domain
}
Response: {
  solution: UserSolution
}
```

### 保存方案
```
POST /api/solutions
Request: UserSolution
Response: {
  id: string,
  solution: UserSolution
}
```

### 更新方案
```
PUT /api/solutions/:id
Request: Partial<UserSolution>
Response: UserSolution
```

### 获取方案详情
```
GET /api/solutions/:id
Response: UserSolution
```

### 生成分享链接
```
POST /api/solutions/:id/share
Response: {
  shareUrl: string,
  expiresAt: string
}
```

## 响应式设计

### 移动端 (<768px)
- 单列布局
- 工具列表横向滚动
- 步骤列表垂直堆叠
- 底部操作栏固定

### 平板 (768px-1024px)
- 工具列表网格布局（2列）
- 步骤列表可保持时间轴样式

### 桌面 (>1024px)
- 最大宽度：1000px，居中显示
- 工具列表网格布局（3列）
- 步骤列表时间轴样式
- 右侧可添加固定导航（步骤快速跳转）

## 动画规范
- 加载状态：旋转动画 + 文字淡入淡出
- 方案生成完成：从上淡入 + 轻微放大（scale 0.95 → 1）500ms
- 步骤展开/折叠：slide-down/up 300ms
- 保存成功：按钮变为绿色勾选图标，持续1秒

## 交互反馈
- 生成完成：Medium Haptic + Success Toast
- 保存成功：Success Haptic + Toast:"已保存"
- 复制链接：Light Haptic + Toast:"链接已复制"
- 步骤勾选：Light Haptic

## AI生成逻辑
- **输入**：工具ID列表 + 用户目标（可选）+ 领域
- **处理**：
  1. 分析每个工具的特性和适用场景
  2. 识别工具之间的协作关系
  3. 根据领域和工具特性生成标题
  4. 生成3-8个逻辑清晰的步骤
  5. 为每个步骤标注涉及的工具
  6. 估算每个步骤的耗时
  7. 生成额外建议和提示
- **输出**：完整的UserSolution对象

## 性能优化
- 生成方案时显示流式输出（如果API支持）
- 保存方案防抖（避免重复点击）
- 图片懒加载
- Markdown渲染缓存

## 错误处理
- AI生成失败：
  - 显示错误提示："生成方案失败，请稍后重试"
  - 提供重试按钮
  - 保留用户选择的工具
- 保存失败：
  - Toast提示："保存失败，请检查网络"
  - 保留方案内容，允许重新保存
- 网络超时：
  - 30秒超时提示
  - 允许用户继续等待或取消

## 埋点需求
- 方案生成成功率
- 生成耗时统计
- 工具组合频率分析
- 方案保存率
- 分享转化率
- 步骤完成勾选行为
