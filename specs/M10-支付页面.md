# M10 支付页面规范

## 页面概述
- **路由**: `/payment` 或 `/pricing`
- **优先级**: Phase 3（价值交换机制）
- **核心理念**: 价值交换 - 付费或投稿
- **设计重点**: 透明、友好、无压力

## 功能模块

### 1. Header
- **返回按钮**
  - 图标：左箭头
  - 点击：返回上一页
- **标题**
  - "价值交换" 或 "选择您的贡献方式"

### 2. 价值主张区域
- **标题**
  - "选择您的贡献方式"
  - 字号：xl-2xl，居中
- **描述**
  - "盘根采用「价值交换」机制，您可以选择以下任一方式继续使用："
  - 字号：sm-base，灰色
  - 居中对齐
- **试用期提示（条件显示）**
  - 显示条件：用户处于试用期
  - 内容："您还有 X 天试用期"
  - 样式：浅蓝色背景卡片

### 3. 选择方式卡片

#### 3.1 付费支持卡片
- **图标**
  - 💰 或钱包图标
- **标题**
  - "付费支持"
  - 字号：lg-xl，加粗
- **价格**
  - 月付：¥19.9/月
  - 年付：¥199/年（省 ¥39.8）
  - 价格字号：2xl-3xl
  - 年付标注"推荐"徽章
- **权益说明**
  - ✓ 无限制使用所有功能
  - ✓ AI智能方案生成
  - ✓ 优先获得新功能
  - ✓ 支持网站持续运营
  - ✓ 随时可申请退款
- **选择按钮**
  - 主按钮样式（蓝色渐变）
  - 文字："选择此方式"
  - 点击：展开支付方式选择

#### 3.2 内容投稿卡片
- **图标**
  - ✍️ 或笔图标
- **标题**
  - "内容投稿"
  - 字号：lg-xl，加粗
- **要求**
  - 每月贡献 1 篇优质内容
  - 或每季度贡献 3 篇优质内容
- **权益说明**
  - ✓ 通过创作获得使用权
  - ✓ 内容将展示在平台上
  - ✓ 获得创作者认证标识
  - ✓ 优先推荐您的文章
  - ✓ 成为社区贡献者
- **选择按钮**
  - 次要按钮样式（边框样式）
  - 文字："选择此方式"
  - 点击：跳转到投稿指南或编辑器

### 4. 支付方式选择（展开区域）
- **显示条件**
  - 用户点击"付费支持"的[选择此方式]按钮后
- **周期选择**
  - Tab切换：月付 / 年付
  - 年付显示优惠信息："立省 ¥39.8"
- **支付渠道**
  - 微信支付
    - 图标：微信图标
    - 文字："微信支付"
    - 点击：生成二维码或拉起微信支付
  - 支付宝
    - 图标：支付宝图标
    - 文字："支付宝"
    - 点击：生成二维码或拉起支付宝
- **支付按钮**
  - 主按钮样式
  - 文字："立即支付 ¥19.9" 或 "立即支付 ¥199"
  - 点击：调用支付接口

### 5. 支付流程

#### 5.1 二维码支付（桌面端）
- **流程**
  1. 点击支付按钮 → 生成二维码
  2. 显示二维码模态框
  3. 用户扫码支付
  4. 轮询支付状态（每2秒）
  5. 支付成功 → 跳转到支付成功页
- **二维码模态框内容**
  - 二维码图片（300x300px）
  - 提示文字："请使用微信/支付宝扫码支付"
  - 订单金额显示
  - 倒计时：5分钟（过期需重新生成）
  - 取消按钮

#### 5.2 H5支付（移动端）
- **流程**
  1. 点击支付按钮 → 拉起微信/支付宝APP
  2. 用户在APP内完成支付
  3. 返回应用 → 查询支付状态
  4. 支付成功 → 跳转到支付成功页

### 6. 支付成功页
- **路由**: `/payment/success`
- **内容**
  - ✅ 图标（大号，绿色）
  - 标题："感谢您的支持"
  - 感谢文案：
    ```
    这笔钱将用于网站的维护与壮大，
    让更多人受益于AI工具的力量。
    
    如果您感觉无用，
    请联系我们退款，我们将在24小时内处理。
    ```
  - 订单信息
    - 订单号：xxxxxx
    - 支付金额：¥19.9
    - 有效期至：2025-xx-xx
  - 操作按钮
    - [开始使用]（主按钮）：跳转到首页
    - [查看订单]：跳转到个人中心-订单页
- **自动化**
  - 支付成功后自动开通会员权限（后端处理）
  - 发送邮件通知（可选）

### 7. 投稿指南页面（独立页面）
- **路由**: `/contribute`
- **内容**
  - 投稿要求
    - 内容质量标准
    - 字数要求（>800字）
    - 原创性要求
    - 格式要求（Markdown）
  - 投稿流程
    1. 编写文章（使用Markdown编辑器）
    2. 提交审核
    3. 管理员审核
    4. 审核通过 → 发布 + 开通权限
    5. 审核不通过 → 修改后重新提交
  - 注意事项
    - 每月至少投稿1篇
    - 未完成投稿将自动转为试用期到期
  - 操作按钮
    - [开始创作]：跳转到Markdown编辑器
    - [查看示例文章]

### 8. 退款说明区域
- **位置**
  - 支付方式选择下方
  - 或独立的FAQ区域
- **内容**
  - **退款承诺**
    - "无收获可随时退款"
  - **退款流程**
    1. 进入"个人中心 → 设置 → 退款申请"
    2. 填写退款原因（可选）
    3. 提交申请
    4. 24小时内人工审核
    5. 审核通过后1-7个工作日到账
  - **退款政策**
    - 使用超过30天后不支持退款
    - 已发布投稿的创作者不可退款

### 9. FAQ区域（可选）
- **常见问题**
  - Q: 付费后可以使用多久？
  - A: 月付1个月，年付1年，到期后可续费或转为投稿方式。
  
  - Q: 投稿后审核需要多久？
  - A: 通常24-48小时内完成审核。
  
  - Q: 投稿不通过怎么办？
  - A: 可根据反馈修改后重新提交。
  
  - Q: 可以随时切换付费和投稿方式吗？
  - A: 可以，但已支付费用不退还。

### 10. Footer
- **内容**
  - 用户协议
  - 退款政策
  - 联系客服（邮箱或微信）

## 状态管理

```typescript
interface PaymentPageState {
  // 用户信息
  user: {
    membershipStatus: 'trial' | 'paid' | 'contributor' | 'expired';
    trialDaysLeft?: number;
    membershipExpiresAt?: string;
  };
  
  // 选择方式
  selectedMethod: 'payment' | 'contribution' | null;
  
  // 付费相关
  paymentPlan: 'monthly' | 'yearly';
  paymentChannel: 'wechat' | 'alipay' | null;
  
  // 支付流程
  showPaymentModal: boolean;
  qrCodeUrl: string;
  orderId: string;
  orderAmount: number;
  paymentCountdown: number; // 二维码倒计时（秒）
  isPollingPayment: boolean; // 是否正在轮询支付状态
  
  // UI状态
  isProcessing: boolean;
}

interface Order {
  id: string;
  userId: string;
  amount: number;
  plan: 'monthly' | 'yearly';
  channel: 'wechat' | 'alipay';
  status: 'pending' | 'paid' | 'expired' | 'refunded';
  qrCodeUrl?: string;
  createdAt: string;
  paidAt?: string;
  expiresAt: string; // 会员到期时间
}
```

## API接口

### 创建订单
```
POST /api/payment/create-order
Request: {
  plan: 'monthly' | 'yearly',
  channel: 'wechat' | 'alipay'
}
Response: {
  orderId: string,
  amount: number,
  qrCodeUrl?: string, // 桌面端二维码
  payUrl?: string, // 移动端支付链接
  expiresAt: string // 订单过期时间
}
```

### 查询支付状态
```
GET /api/payment/order/:orderId/status
Response: {
  status: 'pending' | 'paid' | 'expired',
  paidAt?: string,
  membershipExpiresAt?: string
}
```

### 申请退款
```
POST /api/payment/refund
Request: {
  orderId: string,
  reason?: string
}
Response: {
  refundId: string,
  status: 'pending' | 'approved' | 'rejected',
  estimatedDays: number
}
```

### 提交投稿申请
```
POST /api/contribution/apply
Response: {
  success: boolean,
  message: string
}
```

## 响应式设计

### 移动端 (<768px)
- 单列布局
- 方式卡片垂直堆叠
- 使用H5支付（拉起APP）
- 价格信息简化显示

### 平板 (768px-1024px)
- 方式卡片网格布局（2列）
- 支持二维码支付

### 桌面 (>1024px)
- 最大宽度：900px，居中
- 方式卡片并排显示（2列）
- 二维码支付（模态框）

## 动画规范
- 页面加载：fade-in 500ms
- 卡片选择：边框高亮 + 轻微放大 200ms
- 支付方式展开：slide-down 300ms
- 二维码生成：fade-in + scale 300ms
- 支付成功：✅图标从小到大 + 旋转 500ms

## 交互反馈
- 选择方式：Light Haptic
- 点击支付：Medium Haptic
- 支付成功：Success Haptic + Toast
- 二维码过期：Warning Toast

## 支付安全
- HTTPS传输
- 订单签名验证
- 二维码5分钟过期
- 支付状态加密
- 防重复提交

## 轮询策略
- 支付状态轮询：每2秒查询一次
- 最多轮询150次（5分钟）
- 超时提示："支付超时，请重新生成订单"

## 错误处理
- 订单创建失败：Toast提示 + 重试按钮
- 支付超时：提示重新支付
- 网络错误：显示错误信息 + 刷新按钮
- 二维码加载失败：显示错误提示 + 重新生成

## 业务逻辑

### 权限开通逻辑
- 支付成功后：
  1. 后端接收支付回调
  2. 验证签名
  3. 更新订单状态为`paid`
  4. 更新用户会员状态
  5. 设置会员到期时间
  6. 发送通知（可选）

### 投稿审核逻辑
- 用户提交投稿：
  1. 文章进入待审核状态
  2. 管理员审核（M14后台）
  3. 审核通过：
     - 文章发布到平台
     - 更新用户贡献计数
     - 开通会员权限（或延长有效期）
  4. 审核不通过：
     - 通知用户修改原因
     - 用户可重新提交

## 埋点需求
- 页面访问量（PV/UV）
- 方式选择分布（付费 vs 投稿）
- 支付方式选择（微信 vs 支付宝）
- 付费周期选择（月付 vs 年付）
- 支付转化率（各环节漏斗）
- 退款申请率
- 投稿申请数量
- 支付失败原因分析
