## 二、访问控制

### 2.1 三层访问控制模型
```
Level 1: 领域权限
  - creative/dev/work → 公开
  - life → 需学生认证

Level 2: 学校隔离（仅 life 领域）
  - content.schoolId === cert.schoolId → 可见
  - 不匹配 → 完全过滤（不显示）

Level 3: 内容权限（仅 creative/dev/work）
  - visibility: 'public' → 公开
  - visibility: 'campus' → 需认证 + 学校匹配
```

### 2.2 访问控制函数
```typescript
// src/utils/access.ts

export function canAccessDomain(domain: Domain, cert: StudentCertification): boolean {
  const config = DOMAIN_CONFIG[domain];
  if (!config.requiresAuth) return true;
  return cert.status === 'verified';
}

export function canAccessContent(
  content: { domain: Domain; visibility?: string; schoolId?: string },
  cert: StudentCertification
): boolean {
  // Level 1
  if (!canAccessDomain(content.domain, cert)) return false;
  
  // Level 2
  if (content.domain === 'life') {
    return !!cert.schoolId && cert.schoolId === content.schoolId;
  }
  
  // Level 3
  if (content.visibility === 'campus') {
    return cert.status === 'verified' && cert.schoolId === content.schoolId;
  }
  
  return true;
}
```

- 只显示 `content.schoolId === cert.schoolId` 的内容
- 其他学校的内容完全不可见（过滤掉）

## 四、路由与导航

| 场景 | 行为 |
|------|------|
| 未认证点击「生活专区」 | 切换领域成功，但首页显示 Empty State |
| 已认证切换到「生活专区」 | 切换成功，显示本校内容 |
| `/campus` 页面 | 保留，展示 creative/dev/work 中 visibility=campus 的聚合 |
