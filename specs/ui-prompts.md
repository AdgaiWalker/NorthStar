## 设计系统规范（每个提示词前都加）

```
中文 Web 应用。设计规范如下：

【品牌】
- 品牌名：盘根·AI 指南针（PanGen AI Compass）
- 校园站品牌名：北星校园
- 主色渐变：#2563EB → #4F46E5（蓝→靛蓝）
- 校园站渐变：#2563EB → #06B6D4（蓝→青）

【视觉风格：通透质感】
- 毛玻璃效果仅做点缀，用于 Header、浮动操作栏、弹窗遮罩层等少量关键区域
- 毛玻璃参数：低透明度(bg-white/70~80) + 中等模糊(backdrop-blur-md)，保持内容可读性，禁止过度模糊导致"糊感"
- 卡片和内容区域使用实色背景(#fff)，不滥用毛玻璃
- 渐变光影作为分层手段：不同层级卡片用微妙的阴影梯度区分深度，而非堆叠透明度
- 层次感来源：阴影梯度(shadow-sm → shadow-md → shadow-lg) + 边框色深浅 + 间距节奏，而非大面积半透明
- 高级质感靠细节克制：统一圆角体系、统一间距倍数、统一的阴影梯度，而非视觉装饰

【颜色】
- 页面背景：#f8fafc / 卡片：#fff / 边框：#e2e8f0
- 主文字：#1e293b / 副文字：#64748b / 弱文字：#94a3b8
- 领域色：影视创作=purple / 编程开发=blue / 通用办公=green
- 校园 8 分类色：新生报到=#3B82F6 吃=#F59E0B 买=#8B5CF8 出行=#10B981 办事=#6366F1 活动=#EC4899 二手=#F97316 避坑=#EF4444

【字体】Inter, system-ui, sans-serif
【圆角】卡片 12-16px / 按钮 12px / 胶囊 full / 输入框 12px
【阴影层级】
- 层级1(平面)：shadow-sm — 默认卡片、标签
- 层级2(微浮)：shadow-md — 卡片悬停、输入框 focus
- 层级3(浮起)：shadow-lg — 下拉菜单、弹窗、浮动操作栏
- 层级4(顶层)：shadow-xl — 模态框、全屏遮罩内容
- 悬停动效：上移 1px + 阴影升至下一层级
【动效】所有过渡 300ms ease
【最大宽度】max-w-7xl (80rem)
【Header】sticky，低透毛玻璃(bg-white/80 backdrop-blur-md)，高度 64px
【主题】支持标准模式和护眼模式（暖色调 slate→stone）

【信息层级规则】
- 每个页面只有 1 个视觉焦点（最大标题或主 CTA）
- 次级信息通过字号阶梯(text-5xl → text-2xl → text-base → text-sm → text-xs)自然分层
- 装饰性元素(渐变、光效)不遮挡可读内容
- 密集数据区(列表/表格)减少装饰，保持信息密度
- 稀疏区域(Hero/空状态)可适当增加视觉丰富度

【交互规则】
- 所有卡片悬停必须上移 1px + 阴影升至下一层级
- 所有可点击元素必须有 hover 状态变化
- 所有按钮点击必须有 active:scale-95 反馈
- 加载状态必须有 loading 动画
- 空状态必须有引导文案和操作入口
- 表单提交必须有成功/失败/加载三种状态反馈
- 删除操作必须二次确认
```

## 一、全球站（com）

### 1. 全球站首页

```
全球站首页。品牌：盘根·AI 指南针，定位"全球 AI 知识平台"。

【内容数据】
- 3 个领域：影视创作(purple) / 编程开发(blue) / 通用办公(green)
- 工具：封面图 + 名称 + 描述 + 评分 + 使用量 + 领域标签
- 专题：封面图 + 标题 + 描述 + 文章数
- 文章：封面图 + 标题 + 摘要 + 作者 + 日期 + 阅读时长 + 浏览/点赞/评论数 + 领域标签
- 资讯：标题 + 摘要 + 来源 + 时间，支持文字/图片/视频三种媒体形式

【UX 交互】
- 领域切换：下拉菜单切换三个领域，切换后所有内容区域联动过滤（工具/专题/文章/资讯）
- 搜索框：支持 AI 搜索和普通搜索两种模式切换，AI 模式搜索框有蓝色发光效果
- AI 搜索输入后跳转搜索结果页
- 工具卡片：可点击查看详情，有"加入方案"按钮，点击后按钮变为已选状态(ring-2 blue)
- 专题卡片：可点击进入专题页
- 文章卡片：可点击进入文章详情页
- "跳转校园站"链接：新标签页打开，不传递登录态
- 资讯卡片：视频类型有播放按钮图标覆盖在封面上

【响应式】
- 移动端 Header 收起为汉堡菜单
- 移动端内容网格从多列变为单列或双列
- 搜索框全宽
```

### 2. 全球站文章详情页

```
文章详情阅读页。

【内容数据】
- 文章：封面图 + 标题 + 作者信息(头像/名字/日期/阅读时长) + Markdown 正文 + 领域标签 + 关联工具(可选) + 验证徽章(可选)
- 专题目录：当前专题的文章列表，当前文章高亮

【UX 交互】
- 专题目录：点击其他文章切换阅读，当前文章有蓝色竖条高亮
- 文章目录(TOC)：跟随滚动自动高亮当前阅读位置，点击跳转到对应章节
- 底部互动：点赞(可取消)、收藏(可取消)、分享(复制链接到剪贴板，成功提示)
- 关联工具卡片：点击跳转工具详情
- 验证徽章：显示"实测有效"绿色标签，hover 显示验证方式和时间
- 桌面端三栏：专题目录 + 正文 + TOC

### 3. 全球站搜索结果页

```
搜索结果页，支持普通搜索和 AI 搜索。

【内容数据】
- AI 搜索结果：摘要文字 + 路径建议 + 推荐工具(≤3) + 推荐文章(≤2)
- 普通搜索结果：工具/文章/专题/资讯四种类型
- 无结果状态：提示文案 + 相似推荐 + 需求收集入口

【UX 交互】
- 搜索模式切换：AI/普通，切换后重新搜索
- AI 结果区：独立视觉风格区分于普通结果，有 AI 图标标识和角色名("AI 软件顾问")
- 推荐工具可"加入方案"，推荐文章可点击阅读
- AI 结果底部"生成方案"按钮：仅当选了工具时可用
- 普通搜索：Tab 切换 全部/工具/文章/专题/资讯，切换后过滤结果
- 无结果：显示引导搜索其他关键词 + "告诉我你想了解什么"需求收集输入框
- 结果分页：底部分页器
```

### 5. 全球站方案生成页

```
方案生成页，产品核心转化页。

【交互流程】
Step 1 选择工具 → Step 2 输入目标 → Step 3 生成方案 → 展示结果 → 反馈

【UX 交互】
- 工具选择：点击选中/取消，选中状态有 ring-2 blue 边框，已选数量实时计数
- 目标输入：textarea，下方有 3 个示例目标标签可点击自动填充
- "生成方案"按钮：渐变背景 + 发光效果，下方提示"本次消耗 1 次 AI 额度"
- 生成过程：3s loading 动画
- 结果展示：方案标题(渐变文字) + AI 建议(多步骤，每步有序号)
- 保存方案：主按钮，保存后进入反馈流程
- 导出：下拉菜单选择格式(md/txt/csv)，点击下载文件
- 重新生成：回到目标输入步骤
- 反馈："这个方案有用吗？"两个按钮 有用/没用，点"没用"展开文本框输入补充反馈
```

```

【Tab 切换】文章 / 专题 / 资讯 / 工具(com站)

```
认证审核页。

【Tab 切换】com 站注册申请 / cn 站学生认证

【com 站注册申请】

- 批准：绿色按钮 → 确认弹窗 → 系统自动发邮件

### 15. 管理后台 - 内容分析

```
AI 内容分析页。

【三种分析模式(大卡片选择)】
- 内容质量分析(quality)：分析指定内容的互动率、评论情感、参与趋势
- 用户需求分析(demand)：分析搜索和行为趋势，发现热门主题和内容缺口
- 创作方向建议(direction)：基于需求分析给出优先级排序的创作建议

【交互】
- quality：勾选分析对象 + 勾选分析维度 → 开始分析 → 展示 AI 摘要 + 雷达图 + 高光/警告列表 → 导出(pdf/word)
- demand：选时间范围+领域 → 开始分析 → 热门主题 TOP10 + 内容缺口 + 情绪分布 → 导出
- direction：自动基于 demand 结果 → 优先级排序列表(高需求低供给=高优先级红色标记) → 每条建议含主题+形式+理由
```

### 16. 管理后台 - AI 引擎配置

```
AI 引擎配置页。

【内容】
- 当前模型状态：模型名 + 连接状态(绿色/红色) + "测试连通性"按钮
- 模型参数：模型选择下拉 + 温度滑块(0-1) + 最大 token + 超时时间
- 额度配置：免费/Pro 用户各自的 AI 搜索次数和方案生成次数
- 用量统计：今日调用量 / 本月调用量 / 本月费用 + 近 7 天趋势折线图

【交互】
- 测试连通性：点击 → loading → 成功(绿色) / 失败(红色错误信息)
- 所有配置修改后需"保存"按钮确认
- 额度配置：数字输入框，保存后立即生效
```

## 四、流程图

### 17. 方案生成完整链路

```
用户流程图。展示方案生成从搜索到保存导出的完整链路：

### 18. 双站知识流闭环
