# 盘根最终全量实现方案

> 日期：2026-04-25  
> 目标：只描述盘根最终应该成为的样子，不按当前工作量、MVP 或现有代码限制裁剪。  
> 审理对象：`specs/MISSION.md`、`specs/PRD-盘根校园-v9.md`、`specs/PRD-盘根AI指南针-标准版.md`、`specs/架构设计.md`、`.specify/memory/constitution.md`、当前 `frontlife-web` / `frontai-web` / `server` 代码形态。  
> 输出性质：目标态裁定 + 全量实现蓝图。具体任务拆分仍由 `specs/全量任务计划.md` 承接。

## 0. 审理结论

盘根最终不是两个孤立网站，也不是一个通用内容站复制出两个皮肤。盘根最终是一套统一平台能力，承载两个产品：

- 校园站 `frontlife-web`：可信校园知识网络。核心句是：**搜一下，可信的。**
- 全球站 `frontai-web`：AI 实践方案生成器。核心句是：**说一声，方案给你。**
- 共享后端 `server`：统一账号、能力、额度、AI 网关、审核、通知、数据、洞察和审计。
- 后台 `admin-console`：运营、审核、配置、开放、洞察、报告和回滚控制台。

最终信息架构必须服务两个不同核心，不追求两站页面、底部导航、内容模型的对称。

### 0.1 最高优先级裁定

1. **按当前目标架构采用统一账号。**  
   `specs/架构设计.md` 已裁定统一账号、账号级 LV 两站共享，校园 profile、全球 profile、知识库和个人行为数据分开。该裁定也写入当前 `AGENTS.md` 项目约束。

2. **必须修订上层文档冲突。**  
   `.specify/memory/constitution.md` 中仍存在“两站用户账号体系完全独立”等旧口径。最终态不能保留这个冲突。全量实现的第 0 项是同步宪法/使命/PRD 的账号与数据边界口径：
   - 账号身份统一。
   - 账号等级共享。
   - 两站 profile 分离。
   - 两站业务数据分离。
   - 跨域 Cookie/token 不直接共享。
   - cn 原始个人数据不流向海外。

3. **校园站不以 AI 为主角。**  
   AI 是搜索兜底、摘要、分类、写作辅助和洞察工具。用户看到的是可信答案、空间、文章、帖子和反馈。

4. **全球站以方案为核心入口。**  
   全球站不是普通工具目录，也不是资讯站。工具、专题、文章、资讯都服务于“输入目标，生成可执行方案”。

5. **AI 不替人做最终决策。**  
   AI 可以生成答案、草稿、摘要、洞察、建议；不能自动发布、自动精选、自动下架、自动改变产品规则。

6. **所有最终能力必须可控开放。**  
   不是因为功能完成就全部上线。所有非核心能力经 feature flag、阶段开放、人群开关、额度、成本熔断控制。

## 1. 最终产品结构

```text
PanGen Platform
├─ frontlife-web        校园站：可信校园知识网络
├─ frontai-web          全球站：AI 实践方案生成器
├─ admin-console        统一后台：运营 / 审核 / 配置 / 洞察 / 报告
├─ server               共享后端：identity / campus / compass / platform
└─ shared               纯 TypeScript 契约、类型、常量、校验与敏感词能力
```

### 1.1 两站关系

| 维度 | 校园站 `xyzidea.cn` | 全球站 `xyzidea.com` |
| --- | --- | --- |
| 产品定位 | 可信校园信息搜索、沉淀、更新 | AI 工具与实践方案生成 |
| 用户心智 | 搜一下，可信的 | 说一声，方案给你 |
| AI 角色 | 隐形辅助 | 显性能力 |
| 内容核心 | 空间、文章、帖子、反馈 | 工具、专题、文章、资讯、方案 |
| 信任来源 | 本地内容、有帮助、有变化、空间维护者、更新时间、审核记录 | 平台审核、工具资料完整度、方案保存率、方案有效率、用户反馈 |
| 底部导航 | 首页 / 探索 / 我的 | 首页 / 方案 / 我的 |
| 跨站触点 | 低权重引导去全球站学习 AI | 不反向污染校园站 |
| 数据边界 | 校园 profile 与校园行为数据 | 全球 profile 与方案/收藏/工具行为数据 |

## 2. 校园站最终态

校园站目标：让学生和老师快速得到可信校园答案，并把信息沉淀为空间。

### 2.1 页面架构

校园站最终只保留 6 个主页面：

```text
/
  首页：搜索 + 推荐空间 + 生活流

/search
  搜索结果：本地内容优先 + AI 兜底

/explore
  探索页：空间目录

/space/:id
  空间页：知识体系 + 最新动态

/article/:id
  文章阅读页：深度内容 + 反馈

/me
  我的：贡献、空间、内容、收藏、设置
```

帖子详情、写作、回复、通知详情、空间认领等不新增主页面，使用 overlay、抽屉或当前页面内嵌面板承载。

### 2.2 首页

首页承担“问”和“看”的入口：

- 搜索框 placeholder 固定为“问点什么，或者分享点什么”。
- 4 张推荐空间卡片，展示真实空间名和确认数。
- 生活流包含文章更新、帖子、变动提醒、AI 回答来源提示。
- 移动端首页搜索框下方提供“写点什么”展开输入区域，仅注册用户可见。
- 桌面端顶栏提供搜索框和“写点什么”入口。
- 不使用 SpotlightBar。
- 不造假确认数、浏览数、收藏数。

### 2.3 搜索结果

搜索结果页有三种模式，统一遵循“本地可信内容优先”：

1. 本地精确匹配：展示文章/空间/帖子结果，高亮相关段落；AI 回答折叠在下方。
2. 本地部分匹配：本地内容排在前面，AI 作为补充。
3. 本地无匹配：AI 流式回答，标注“由 AI 生成，仅供参考”，并提供“我知道准确信息，来补充”。

搜索无结果不能成为死路：

- 推荐相关已有内容。
- 提供发起求助入口。
- 将 query 记录为搜索缺口。
- 进入中台 insight，用于驱动内容任务。

### 2.4 探索页

探索页是空间目录，不是普通分类页：

- 展示所有空间的网格或列表。
- 支持搜索过滤。
- 每张卡片展示 Lucide 图标、名称、描述、确认数、文章数、最近活跃时间。
- 排序优先考虑活跃度、时效性、内容完整度。
- 不展示内部等级、算法细节。

### 2.5 空间页

空间是校园站的核心知识容器：

- 空间头：图标、名称、描述、确认数、文章数、维护者。
- 知识体系在上，最新动态在下。
- 不做“文章 / 讨论”Tab。
- 知识区展示系统性文章，层级最多两层。
- 动态区展示帖子和变动提醒。
- 注册用户看到底部输入框：“在这里说点什么…”。
- 输入内容自动归属当前空间。
- 未选择空间的首页短帖默认归属“自由广场”。

### 2.6 文章阅读页

文章页承担深度内容和信任反馈：

- 桌面端：左侧空间文章树，中间正文和目录，右侧辅助信息。
- 移动端：单栏，顶部折叠目录。
- 底部固定两个反馈动作：有帮助 / 有变化。
- “有变化”是公开信息，显示在文章标题附近。
- 支持上一篇 / 下一篇。
- 文章读完后可以低权重出现全球站触点：“想了解 AI 是怎么做到的？”

### 2.7 我的页

我的页显示贡献，不显示内部等级：

- 帮助了 X 人。
- 写了 X 篇。
- X 个收藏。
- 我的空间。
- 我的内容。
- 我的收藏。
- 通知入口。
- 设置、协议、隐私、数据导出、注销申请。
- 资深用户才显示“创建空间”按钮，但不展示“LV3”标签。

### 2.8 校园站移动端导航

最终移动端底部导航固定三项：

```text
首页     探索     我的
```

裁定：

- `/` 高亮首页。
- `/search` 可高亮首页，因为搜索是首页主动作的结果。
- `/article/:id` 可高亮首页或按来源回显；默认高亮首页。
- `/explore`、`/space/:id` 高亮探索。
- `/me` 高亮我的。
- 登录页、后台页不显示底部导航。
- 不出现浮动加号按钮。
- 写作入口不进入底部一级导航。

## 3. 全球站最终态

全球站目标：帮助用户主动掌握 AI，围绕目标生成可执行方案。

### 3.1 页面架构

```text
/
  首页：目标入口 + 领域切换 + 工具 / 专题 / 文章 / 资讯

/tool/:toolId
  工具详情：能力、场景、用法、案例、收藏、加入方案

/article/:articleId
  文章阅读：教程、案例、专题内容

/solution/new
  新建方案：输入目标，可选工具，生成方案

/solution/:solutionId
  方案详情：步骤、工具、导出、反馈、继续优化

/me
/me/profile
/me/solutions
/me/favorites
/me/credits
/me/settings
  用户中心

/login
  登录注册

/admin/*
  后台
```

### 3.2 首页

首页不是营销页，第一屏直接进入可用体验：

- 主入口是目标输入或 AI 搜索。
- 用户可以切换领域：影视创作、编程开发、通用办公。
- 展示工具推荐、专题、文章、资讯。
- 工具卡片支持收藏和加入方案。
- 精选工具可以使用沉浸式展示，但不得压过“目标 -> 方案”主路径。
- 首页选中工具后出现 `FloatingDock`，作为快捷生成方案入口。

### 3.3 方案页

方案页是全球站核心页面。最终体验必须目标优先：

- `/solution/new` 永远可进入。
- 用户不需要先选工具。
- 有已选工具时：基于已选工具和目标生成方案。
- 无已选工具时：用户先输入目标，系统推荐工具，再生成方案。
- 工具选择是辅助，不是门槛。
- 额度不足时进入 Degraded 态，提供不消耗额度的基础草稿或明确提示。
- 生成结果必须标注 AI 生成。
- 方案保存后进入 `/solution/:solutionId` 或 `/me/solutions`。

当前 `frontai-web/src/components/MobileBottomNav.tsx` 的三项方向成立，但 `/solution/new` 不能再依赖“先选工具”。

### 3.4 工具详情页

工具详情不是孤立百科：

- 工具定位。
- 适合场景。
- 不适合场景。
- 上手步骤。
- 相关教程和案例。
- 价格/平台/链接等基础信息。
- 实测有效或需复查标记。
- 收藏。
- 加入方案。
- 用该工具生成方案。

### 3.5 文章阅读页

文章页服务实践，不做泛知识堆叠：

- 支持专题目录。
- 支持关联工具。
- 支持从文章中的目标或步骤进入方案生成。
- 过期内容显示“信息可能已变更”。
- 已验证内容展示验证标记。

### 3.6 用户中心

用户中心包含：

- 个人资料。
- 我的方案。
- 收藏夹。
- 我的额度。
- 设置。
- 数据导出和注销入口。

`/me/solutions` 是方案资产库，不只是历史列表：

- 查看方案详情。
- 导出 md/txt/csv，后续可扩展 PDF/Word。
- 删除方案。
- 提交“有用 / 没用”反馈。
- 对没用方案触发补充建议。

### 3.7 全球站移动端导航

最终移动端底部导航固定三项：

```text
首页     方案     我的
```

裁定：

- 全球站不增加“探索”Tab。“探索 / 空间”属于校园站信息架构。
- `/`、`/tool/:id`、`/article/:id` 默认高亮首页。
- `/solution/new`、`/solution/:id` 高亮方案。
- `/me/*` 高亮我的。
- 登录页、后台页不显示底部导航。
- `FloatingDock` 只能作为“已选工具快捷操作”，不能替代方案页主入口。

### 4.2 账号注册与绑定

早期注册采用最小身份：

```text
用户名 + 密码
```

不强制收集：

- 手机号。
- 邮箱。
- 真实姓名。
- 身份证。
- 学号。
- 精确定位。

后置支持：

- 邮箱绑定。
- 手机号绑定。
- GitHub OAuth。
- 校园认证。

### 4.3 等级与能力

`globalLevel` 属于账号级，两站共享。能力由中台按站点映射：

| 等级 | 校园站能力 | 全球站能力 |
| --- | --- | --- |
| Guest | 浏览、本地搜索、少量游客 AI | 浏览公开内容、少量游客 AI |
| LV1 | 发短帖、有帮助、有变化、基础 AI 搜索 | 收藏、少量方案生成 |
| LV2 | 写长文章、AI 写作、本地文档同步、更高 AI 额度 | 保存/导出方案、内容投稿、本地文档同步 |
| LV3 | 创建空间、认领空间、维护空间、处理变化反馈 | 高级内容工作室、工具/专题投稿、更高额度 |
| Admin | 后台与审核能力 | 后台与审核能力 |

前端不直接写死权限。前端统一查询：

```http
GET /api/platform/capabilities?site=campus
GET /api/platform/capabilities?site=compass
```

### 5.1 命名空间

新接口全部走命名空间：

```text
/api/identity/*
/api/platform/*
/api/ai/*
/api/campus/*
/api/compass/*
/api/admin/*
/api/notification/*
/api/compliance/*
```

旧路径只在迁移期保留，不新增旧路径能力，不写长期兼容层。

### 5.2 AI 网关

所有 AI 调用必须经过后端 AI 网关：

- 输入敏感词检查。
- 本地搜索优先。
- capability 检查。
- quota 检查。
- prompt 组装。
- provider 调用。
- 输出过滤。
- fallback reason 标准化。
- token、成本、耗时记录。
- 调用日志审计。

前端不持有任何真实模型 API Key。

### 5.3 搜索服务

搜索服务按站点分域：

- campus search：空间、文章、帖子、反馈和搜索缺口。
- compass search：工具、专题、文章、资讯和方案入口。

统一搜索事件进入 event-store，但原始个人查询不跨站直接共享。

### 5.4 事件系统

必须采集的事件：

- 搜索。
- 无结果搜索。
- AI 兜底。
- 文章阅读。
- 有帮助。
- 有变化。
- 发帖。
- 回复。
- 空间访问。
- 方案生成。
- 方案保存。
- 方案反馈。
- 工具点击。
- 收藏。
- 跨站跳转。

事件用于洞察、报表、内容任务和产品改进，不用于出售个人数据。

## 8. 核心业务流

### 8.1 校园搜索闭环

```text
用户搜索
-> campus search 本地检索
-> 有足够可信结果：返回本地内容
-> 结果不足：检查 capability / quota / sensitive
-> AI 生成参考回答
-> 标注“由 AI 生成，仅供参考”
-> 用户点击文章、空间或“我知道准确信息，来补充”
-> 有帮助 / 有变化 / 求助帖进入校园反馈系统
-> 事件进入 event-store
-> insight-engine 聚类高频缺口
-> 后台生成内容任务或通知维护者
```

### 8.2 校园空间维护闭环

```text
空间内容被阅读
-> 用户点击有帮助
-> confirmationCount 增加
-> 用户点击有变化
-> 生成公开变化标记和审核任务
-> 审核通过后通知维护者
-> 维护者更新内容
-> 文章更新时间和确认时间刷新
```

### 8.3 全球方案闭环

```text
用户进入 /solution/new
-> 输入目标
-> 可选选择工具
-> 无工具时系统推荐候选工具
-> 检查 capability / quota / sensitive
-> AI 生成方案
-> 用户保存方案
-> 用户使用后反馈有用 / 没用
-> 方案有效率进入北极星指标
-> 没用反馈触发补充建议和内容改进信号
```

### 8.4 跨站自然转化

```text
校园站 AI 回答卡片
-> 低权重“由 AI 生成”说明
-> 低权重链接：想了解更多 AI 工具？
-> 新标签页打开全球站
-> 只记录聚合来源
-> 不传 token
-> 不传个人身份
```

允许触点：

- 校园站 AI 回答卡片。
- 校园站文章阅读完成后。
- 校园站中用户主动讨论 AI 工具时的内容关联。

禁止触点：

- 首页强推广。
- 搜索输入过程打断。
- com 内容自动反向灌入 cn。
- 携带跨站登录态。

## 9. 后台最终态

后台是控制台，不只是 CRUD。

### 9.1 模块

- 用户管理。
- 站点 profile 查看。
- 校园空间与内容管理。
- 全球工具、专题、资讯、文章管理。
- 审核队列。
- 举报处理。
- AI 输出抽检。
- AI 调用、额度、成本控制。
- 功能开放控制台。
- 通知投递。
- 需求洞察工作台。
- 学校报告工作台。
- 合规处理：协议、隐私、数据导出、注销。
- 审计日志。

### 9.2 功能开放控制台

所有非核心能力使用三态：

```text
Hidden     入口不可见
Locked     入口可见但不可用，给出解释
Degraded   功能可用但范围、质量或额度降低
```

阶段开放：

| 阶段 | 开放内容 |
| --- | --- |
| Phase 0 | 内部测试，只允许测试账号，AI 不对游客开放 |
| Phase 1 | 校园本地搜索开放，游客每天少量 AI |
| Phase 2 | 校园互动开放，注册、短帖、有帮助、有变化、审核队列 |
| Phase 3 | 内容共建开放，LV2 长文章，LV3 空间创建和认领 |
| Phase 4 | 全球站 Beta，工具浏览、方案生成、方案保存 |
| Phase 5 | 正式开放，完整配额、升级策略、后置绑定能力 |

### 9.3 需求洞察工作台

洞察不是图表，而是行动建议：

```text
洞察：过去 7 天“宿舍水压”无结果搜索 43 次
证据：query 聚类 + 有变化反馈 + 回复增长
建议：创建“宿舍水压解决办法”内容任务
对象：校园运营 / 相关空间维护者
优先级：高
```

AI 只生成建议，后台人员确认后才创建任务、通知用户或调整配置。

### 9.4 学校报告工作台

学校报告只使用脱敏聚合数据。

允许：

- 高频需求变化。
- 校园信息覆盖情况。
- 活动和通知触达情况。
- 空间访问趋势。
- 内容更新情况。
- AI 兜底下降趋势。

禁止：

- 个人搜索记录。
- 个人发帖轨迹。
- 个人画像。
- 未公开敏感内容。
- 可反推出个人身份的数据。

## 10. 前端工程边界

### 10.1 `@ns/shared`

`@ns/shared` 只能包含纯 TypeScript：

- 类型。
- API 契约。
- 常量。
- Zod schema 或轻量校验。
- 敏感词工具。
- 站点无关工具函数。

禁止：

- React。
- Zustand。
- DOM API。
- UI 组件。
- 前端路由。

### 10.2 两站 UI 不强行复用

两站可以复用类型、API client、工具函数，但不强行复用页面组件。

原因：

- 校园站是高频、移动、任务型、可信信息界面。
- 全球站是目标、工具、方案、学习型界面。
- 强行复用 UI 会让两个产品都变钝。

### 10.3 图标与视觉

- UI 图标统一 Lucide。
- emoji 只允许出现在用户输入文本中。
- 移动端触控目标不少于 44px。
- Safe Area 不遮挡主操作。
- 文案中文优先。
- AI 标注清晰但不喧宾夺主。

## 11. 当前代码到最终态的关键改造点

本节不是按工作量排序，而是指出现状与最终态的差距。

### 11.1 `frontai-web` 方案入口

当前问题：

- `MobileBottomNav.tsx` 的“方案”指向 `/solution/new`。
- `SolutionNewPage.tsx` 在没有已选工具时要求用户回首页先选工具。
- 这和全球站“说出目标，方案给你”的最终定位冲突。

最终改造：

- `/solution/new` 无条件可进入。
- 无工具时展示目标输入和推荐工具。
- 有工具时复用已选工具。
- 生成方案后保存并进入方案详情。
- `FloatingDock` 只作为快捷入口。

### 11.2 `frontlife-web` 6 页收口

当前校园站已接近 6 页结构，但最终必须保证：

- 帖子无独立详情页。
- 写作无独立主页面。
- 通知详情不新增主页面。
- 空间页不做内容类型 Tab。
- 移动端底部导航始终可见，登录/后台除外。

### 11.3 后端命名空间

当前仍可能存在旧接口路径。最终应统一：

- 校园业务进入 `/api/campus/*`。
- 全球业务进入 `/api/compass/*`。
- 账号进入 `/api/identity/*`。
- 能力、额度、开关进入 `/api/platform/*`。
- AI 进入 `/api/ai/*`。

旧路径完成迁移后删除。

### 11.4 文档冲突修复

最终实现前必须处理：

- 宪法旧账号独立口径。
- 旧 PRD 中手机号/SMS/邮箱必选口径。
- 旧架构会议纪要和旧 dev-plan 残留引用。
- `全球specs.md` 顶部仍引用校园 v8，需同步为 v9 或标记过时范围。

## 13. 最终裁定摘要

1. 校园站最终是可信校园知识网络，不是 AI 广告页。
2. 全球站最终是 AI 实践方案生成器，不是工具目录。
3. 两站共享账号和账号级等级，但不共享业务 profile 和原始行为数据。
4. 校园站底部导航是：首页 / 探索 / 我的。
5. 全球站底部导航是：首页 / 方案 / 我的。
6. `frontai-web` 的方案页必须目标优先，不能要求先选工具。
7. 所有 AI 能力进入后端网关，前端不持有真实 key。
8. 后台是控制系统，必须能审核、开放、回滚、洞察和审计。
9. AI 生成建议，不做最终裁决。
10. 文档冲突必须先修，尤其是统一账号与数据边界口径。
