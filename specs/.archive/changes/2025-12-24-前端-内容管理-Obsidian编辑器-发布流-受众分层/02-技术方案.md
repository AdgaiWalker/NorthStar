# 技术方案：内容管理（本地 mock）+ Obsidian 编辑器 + 草稿→发布流 + 受众分层

版本：v0.1
日期：2025-12-24

## 1. 总体实现策略
系统以“个人博客站可持续内容生产”为目标，采用本地 mock 策略：
- 内容数据源以 localStorage 为主，内置种子内容作为初始数据。
- 统一内容模型（ContentItem），前台展示、校内专区访问控制、后台内容管理均引用同一套字段。
- 编辑器实现 Obsidian 风格三栏布局，重点保证写作与保存稳定。

## 2. 信息架构与路由
### 2.1 前台（用户侧）
- 首页：展示当前领域下“用户可访问”的已发布内容
- 文章详情：按 ID 展示文章，校内内容在此进行直链访问拦截
- 校内专区：已认证用户展示本校校内内容；未认证用户展示锁定态

### 2.2 后台（作者侧）
系统复用现有 `AdminLayout`，新增内容管理模块：
- `/admin/content`：内容列表（草稿/已发布）
- `/admin/content/new`：新建草稿并进入编辑器
- `/admin/content/:contentId/edit`：编辑器

## 3. 数据模型（Data Schema）
### 3.1 基础枚举
- `ContentType = 'article'`
- `ContentStatus = 'draft' | 'published'`
- `ContentVisibility = 'public' | 'campus'`

### 3.2 ContentItem
ContentItem 作为统一内容对象，定义为：
- `id: string`：全局唯一
- `type: ContentType`：内容类型
- `title: string`：标题
- `summary: string`：摘要
- `coverImageUrl: string`：封面图 URL（允许为空字符串）
- `domain: 'creative' | 'dev' | 'work'`：领域
- `status: ContentStatus`：状态
- `visibility: ContentVisibility`：可见性
- `schoolId?: string`：校内内容必须存在且非空
- `schoolName?: string`：用于 UI 展示
- `tags: string[]`：标签
- `relatedToolIds: string[]`：关联工具
- `markdown: string`：正文 Markdown
- `createdAt: string`：ISO 时间
- `updatedAt: string`：ISO 时间
- `publishedAt?: string`：ISO 时间
- `stats: { views: number; likes: number }`：本地 mock 统计

### 3.3 ContentAsset（图片资源）
为支持 Obsidian 式图片粘贴/拖拽，系统引入本地资源表：
- `id: string`
- `mime: string`
- `size: number`
- `dataUrl: string`（base64 data URL）
- `createdAt: string`

图片在 Markdown 中以标准语法引用：
- `![alt](dataUrl)`

系统强制约束：
- 单张图片大小上限固定为一个明确值（在实现中定义常量，例如 300KB）
- 超限时拒绝导入并显示错误文案

## 4. 本地存储（STORAGE_KEYS）
在 `pangen-ai-compass/src/utils/storage.ts` 中新增：
- `pangen.content.items`：ContentItem[]
- `pangen.content.assets`：ContentAsset[]

## 5. Guard（数据校验）
在 `pangen-ai-compass/src/utils/guards.ts` 中新增：
- `isContentVisibility`
- `isContentStatus`
- `isContentItem`
- `isContentItemArray`
- `isContentAsset`
- `isContentAssetArray`

系统在读取本地存储时严格执行：
- 校验通过使用存储值
- 校验失败删除损坏值并回退默认值，同时触发一次性 reset 提示

## 6. Store 设计
新增 `pangen-ai-compass/src/store/useContentStore.ts`：

### 6.1 State
- `items: ContentItem[]`
- `assets: ContentAsset[]`
- `storageResetDetected: boolean`（复用提示模式）

### 6.2 Actions
- `createDraft(): string` 返回新内容 ID
- `updateDraft(contentId, patch)`：更新草稿字段与 updatedAt
- `publish(contentId)`：draft → published，写入 publishedAt
- `unpublish(contentId)`：published → draft，清空 publishedAt
- `delete(contentId)`：删除内容
- `upsertAsset(asset)`：写入图片资源

### 6.3 Selectors
- `getPublishedArticlesByDomain(domain)`
- `getArticleById(contentId)`

## 7. 访问控制（受众分层）
### 7.1 Access Policy 函数
新增统一判定函数：
- 输入：`{ visibility, schoolId }` 与 `studentCertification`
- 输出：`boolean` 是否可访问

规则：
- public：永远允许
- campus：必须满足 studentCertification.status === 'verified' 且 schoolId 完全一致

### 7.2 列表过滤与直链拦截
- 首页/搜索/列表：过滤掉不可访问的 campus 内容
- 详情页：若目标为 campus 且不可访问，渲染锁定态并引导去 `/me/certification`

## 8. Obsidian 风格编辑器（实现要点）
### 8.1 布局
桌面端三栏布局固定：
- 左栏：大纲（基于 `extractDocToc(markdown)`）
- 中栏：编辑器（Markdown）
- 右栏：预览区（复用 `DocRenderer`）

移动端策略：
- 三栏折叠为“编辑/预览”双模式切换
- 大纲以抽屉形式出现

### 8.2 编辑器技术选型
系统采用 CodeMirror 6 作为编辑器内核，实现：
- Markdown 语法高亮
- 快捷键（Ctrl+S 保存、Ctrl+B 加粗、Ctrl+I 斜体）
- 基础工具栏（加粗/斜体/链接/图片/代码块/H1-H3）

### 8.3 实时预览
预览区使用现有 `DocRenderer`，并扩展支持：
- 数学公式（LaTeX）：使用 remark/rehype 插件渲染
- 代码高亮：使用 rehype 插件或高亮库渲染

### 8.4 自动保存
系统实现“2 秒自动保存草稿”机制：
- 编辑器内容变化后标记 dirty
- 定时器在 2 秒窗口内将草稿写入 store 并持久化
- 保存成功后更新“已保存时间”

## 9. 与现有审核工作台的关系
系统保留未来扩展点：
- 内容发布前可以新增“提交审核”动作，将内容转化为 review task
- 初期单人博客站直接发布，不进入审核队列

## 10. 与 PRD 的对齐与修订建议
系统将 PRD 的“内容管理”能力前置到初期版本，并合并“我的创作”与“内容管理”的数据模型。
系统将语言偏好明确为 zh/en 两种，并删除日语/俄语范围。
系统保持领域（domain）与校内（visibility/schoolId）为正交维度，统一在内容模型中表达。
