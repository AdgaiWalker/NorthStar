## 🔴 严重问题

### 1. 技术栈不匹配 PRD 要求

**问题描述:**
- 当前代码缺少后端，只有前端实现
- PRD 要求前后端分离架构，后端使用 Hono.js + Prisma + PostgreSQL
- 当前通过 Vite proxy 直接调用智谱 AI，API Key 暴露在前端配置中

**影响范围:**
- 安全风险：API Key 可能泄露
- 架构不完整：缺少数据持久化、用户认证、权限管理
- 无法实现 PRD 中的用户系统、次数管理、支付等功能

**建议方案:**
- [ ] 创建独立的后端项目 (`backend/`)
- [ ] 使用 Hono.js 搭建 API 服务
- [ ] 配置 Prisma + PostgreSQL 数据库
- [ ] 实现 JWT 认证和权限管理
- [ ] AI 调用移至后端，保护 API Key

---

### 2. 缺少 UI 组件库（Shadcn/ui）

**问题描述:**
- PRD 要求使用 Shadcn/ui 作为 UI 组件库
- 当前代码使用原生 HTML 元素和手写样式
- 缺少统一的组件规范和设计系统

**影响范围:**
- 开发效率低
- UI 一致性难以保证
- 可访问性（a11y）支持不足

**建议方案:**
- [ ] 安装 Shadcn/ui 及其依赖（Radix UI）
- [ ] 配置 `components.json` 和导入路径
- [ ] 重构现有组件使用 Shadcn/ui 组件
- [ ] 建立组件库目录结构 `components/ui/`

---

### 3. 缺少状态管理（Zustand）

**问题描述:**
- PRD 要求使用 Zustand 进行状态管理
- 当前使用 React useState 和 props 传递
- 状态逻辑分散在 App.tsx 中，难以维护

**影响范围:**
- 代码耦合度高
- 状态管理混乱
- 难以扩展和测试

**建议方案:**
- [ ] 安装 Zustand
- [ ] 创建状态管理 store（`stores/` 目录）
- [ ] 将全局状态迁移到 Zustand
- [ ] 按功能模块拆分 store（auth, ui, solution 等）

---

### 4. 缺少数据请求层（TanStack Query）

**问题描述:**
- PRD 要求使用 TanStack Query（React Query）管理服务端状态
- 当前使用 fetch 直接调用，缺少缓存、重试、乐观更新等机制
- AI 请求逻辑耦合在组件中

**影响范围:**
- 数据请求逻辑分散
- 缺少加载状态和错误处理的统一管理
- 无缓存机制，重复请求浪费资源

**建议方案:**
- [ ] 安装 @tanstack/react-query
- [ ] 配置 QueryClientProvider
- [ ] 创建 API 请求层（`api/` 目录）
- [ ] 使用 useQuery/useMutation 重构数据请求

---

### 5. 缺少表单验证（React Hook Form + Zod）

**问题描述:**
- PRD 要求使用 React Hook Form + Zod 处理表单
- 当前表单使用受控组件和手动验证
- 缺少统一的验证规则和错误提示

**影响范围:**
- 表单逻辑冗余
- 验证规则分散
- 用户体验不一致

**建议方案:**
- [ ] 安装 react-hook-form 和 zod
- [ ] 定义表单验证 schema（`schemas/` 目录）
- [ ] 重构登录、注册、方案生成等表单
- [ ] 统一错误提示样式

## 🟡 中等问题

### 6. 项目结构不规范

**问题描述:**
- 所有文件平铺在根目录
- 缺少清晰的目录结构
- 不符合前后端分离的项目组织方式

**当前结构:**
```
1pangen-ai-compass/
├── App.tsx
├── types.ts
├── constants.ts
├── components/
├── services/
├── utils/
└── ...
```

**建议结构:**
```
pangen-ai-compass/
├── frontend/               # 前端项目
│   ├── src/
│   │   ├── components/    # 组件
│   │   ├── pages/         # 页面（替代当前的 views）
│   │   ├── stores/        # Zustand stores
│   │   ├── hooks/         # 自定义 hooks
│   │   ├── api/           # API 请求层
│   │   ├── schemas/       # Zod 验证 schemas
│   │   ├── utils/         # 工具函数
│   │   ├── types/         # TypeScript 类型
│   │   ├── constants/     # 常量
│   │   ├── App.tsx
│   │   └── main.tsx
│   ├── public/
│   ├── package.json
│   └── vite.config.ts
└── backend/               # 后端项目（待创建）
    ├── src/
    │   ├── routes/
    │   ├── controllers/
    │   ├── services/
    │   ├── middlewares/
    │   ├── models/
    │   └── index.ts
    ├── prisma/
    ├── package.json
    └── tsconfig.json
```

**建议方案:**
- [ ] 重构项目目录结构
- [ ] 创建 `frontend/` 和 `backend/` 目录
- [ ] 移动前端代码到 `frontend/src/`
- [ ] 创建后端项目脚手架
