## 2. 产品定义

### 2.1 产品定位

> 一句话定义：面向黑河学院学生与泛 AI 学习者的“工具 × 内容 × 路径”聚合检索与方案生成平台。用户提出真实问题，系统输出可执行路径（工具 + 教程 + 步骤），并将过程沉淀为可复用资产。

### 2.2 产品愿景

盘根 · AI 指南针定位为用户的个人 AI 导航器，持续降低以下四个关键阶段的认知与试错成本：

```
选工具 → 找教程 → 搭路径 → 落方案
```

### 2.3 北极星体验

#### 2.3.1 体验定义

用户输入一个真实问题，系统输出一条**高质量、可执行**的路径（工具 + 资料 + 步骤），并允许一键沉淀为个人资产。

输出结构：

| 组件 | 数量 | 说明 |
| --- | --- | --- |
| 工具推荐 | 1–3 个 | 匹配用户需求的工具集合 |
| 内容推荐 | 1–2 个 | 相关教程或文章 |
| 方案步骤 | 3–5 步 | 可执行的行动路径 |
| 一键沉淀 | - | 保存到「我的方案」，支持导出 |

#### 2.3.2 情绪目标

| 时间节点 | 目标情绪 |
| --- | --- |
| 30 秒内 | 从“焦虑/迷茫”到“确定/可执行” |
| 2 分钟内 | 形成“我已经开始推进”的主观感受 |

### 2.4 核心价值主张

#### 对用户的价值

- **明确路径**：减少工具筛选与学习路径探索时间
- **可复用方案**：降低同类任务的重复成本
- **工作流思维**：形成跨工具组合能力

#### 对平台的价值

- 以工具与内容为**基础资产**
- 以方案沉淀形成**用户留存抓手**
- 以共建机制形成**长期内容供给**

#### 商业原则（实事求是）

- 收费目标以覆盖模型调用与运营成本为底线，不以高利润为导向。
- 主要售卖“AI 使用次数”（次数包）；通过“投稿激励次数”驱动共建。

## 3. 用户研究

### 3.1 用户画像

#### 画像 A：新手执行者

| 维度   | 描述                      |
| ---- | ----------------------- |
| 特征   | 对 AI 工具了解有限，缺少清晰路线      |
| 痛点   | 不知道从哪一步开始，不知道选哪些工具      |
| 需求   | 可执行步骤与最低门槛的工具组合         |
| 成功标准 | 一次提问获得可跟做的步骤，产生第一份可保存方案 |

#### 画像 B：进阶整合者

| 维度   | 描述                  |
| ---- | ------------------- |
| 特征   | 使用过多款 AI 工具，缺少稳定工作流 |
| 痛点   | 工具切换频繁，流程不稳定，复用困难   |
| 需求   | 以目标为中心的工具链，以及可复用模板  |
| 成功标准 | 形成可重复执行的流程，多次复用导出方案 |

#### 画像 C：内容学习者

| 维度   | 描述                |
| ---- | ----------------- |
| 特征   | 以教程学习为主，偏好视频与实战文章 |
| 痛点   | 内容分散，缺少与工具选择的对应关系 |
| 需求   | 内容推荐可直达，内容与工具绑定   |
| 成功标准 | 阅读内容后能回到工具选择并生成方案 |

### 3.2 核心任务（JTBD）

| JTBD   | 任务描述                                       |
| ------ | ------------------------------------------ |
| JTBD-1 | **从问题到工具**：当我遇到具体任务时，我要快速找到最匹配的工具集合，从而立即开始 |
| JTBD-2 | **从工具到步骤**：当我选定工具时，我要得到分步骤路径，从而在最短时间拿到结果   |
| JTBD-3 | **从步骤到复用**：当我完成任务后，我要把流程保存导出，从而在下次任务中直接复用  |

## 4. 产品范围

### 4.1 最终范围（产品形态）

本 PRD 描述产品的**最终形态**：聚合工具与内容，提供普通搜索与 AI 搜索，支持方案沉淀与导出，提供用户后台与系统后台，并通过“AI 次数”形成可持续的共建机制。

> **⚠️ V1 实现范围说明**：V1 为纯前端演示版本，主要实现了以下核心功能：
> - ✅ 首页、搜索、工具卡片、内容阅读
> - ✅ AI 搜索（单模型，通过 Vite proxy 转发）
> - ✅ 工具选择、方案生成、方案保存（本地存储）
> - ✅ 方案导出（Markdown/TXT/CSV）
> - ✅ 领域切换、主题切换（浅色/护眼）
> - ✅ 登录 UI 演示（无真实认证）
> 
> **以下功能计划在后续版本实现**：
> - ❌ AI 次数管理、支付、退款（V2-V3）
> - ❌ 学生认证、校内专区（V2）
> - ❌ 系统后台、内容管理（V3-V4）
> - ❌ 数据分析、用量监控（V2-V3）
> - ❌ 多语言（V2）
> - ❌ 普通搜索、库模式选择（V2）
> - ❌ 投稿与共建（V2-V3）

#### 4.1.1 面向用户的核心能力

| 模块   | 功能点 |
| ---- | ---- |
| 搜索   | 普通搜索（免费）+ AI 搜索（消耗次数） |
| 库选择  | AI 搜索支持「专业库（精选）」/「综合库（全量）」 |
| 领域   | 影视创作 / 编程开发 / 通用办公 |
| 内容   | 精品专区 / 大众广场 / 校内专区（学生认证后解锁） |
| 工具   | 工具详情、收藏、工具组合选择 |
| 方案   | 生成方案、保存、导出（md/txt/csv）、分享 |
| 设置   | 主题、语言、导出偏好 |
| 账号   | 登录/注册、学生认证申请、等级成长 |

#### 4.1.2 面向运营与管理的核心能力

| 模块   | 功能点 |
| ---- | ---- |
| 内容管理 | 上传与编辑（你自建内容）、精选/广场/校内专区管理 |
| 认证与等级 | 学生认证审核、用户等级与经验管理、权益发放 |
| 次数与收费 | 次数包 SKU 配置、定价可配置、订单与退款管理 |
| AI 引擎 | 模型配置、连通性测试、用量监控、审计日志 |

### 4.2 阶段性策略（通过后台开关控制）

为降低早期运营成本，部分能力按阶段启用。

| 阶段 | 策略 | 说明 |
| --- | --- | --- |
| 前期 | 关闭投稿入口 | 以你自建内容为主（种子库） |
| 中期 | 推荐制开放投稿 | 通过质量评估/历史通过率/用户等级授予投稿权限 |
| 后期 | 完整共建闭环 | 投稿、审核、精选、校内专区共建完善 |

### 4.3 明确非目标（当前不做）

- 个性化推荐（复杂排序、RAG 增强）
- 复杂部署体系与监控平台对接（保持最小化）
- 3D 登录与重动效（以功能为主）

## 6. 用户旅程

### 6.1 旅程 1：普通搜索（免费）

```
用户进入首页
    ↓
用户在搜索框输入关键词
    ↓
选择模式：普通搜索
    ↓
系统展示搜索结果（工具/内容分组）
    ↓
用户进入工具详情或阅读页
```

### 6.2 旅程 2：AI 搜索（需要登录 + 消耗次数）

```
用户进入首页
    ↓
用户在搜索框输入问题
    ↓
选择模式：AI 搜索
    ↓
若未登录：引导登录/注册
    ↓
选择库：专业库（精选）/综合库（全量）
    ↓
系统输出：摘要 + 路径建议 + 推荐工具 + 推荐内容
    ↓
用户加入工具/生成方案/保存导出
```

### 6.3 旅程 3：选择工具并生成方案

```
用户点击工具卡片上「加入方案」
    ↓
系统把工具加入已选集合
    ↓
系统展示浮动 Dock
    ↓
用户点击 Dock「生成方案」
    ↓
系统进入方案生成页
    ↓
用户输入目标描述
    ↓
用户点击生成
    ↓
系统生成方案内容
    ↓
用户点击保存
    ↓
系统将方案写入「我的方案」
```

### 6.4 旅程 4：管理方案并导出

```
用户进入「我的」
    ↓
用户进入「我的方案」
    ↓
用户点击某个方案
    ↓
系统打开方案详情抽屉
    ↓
用户点击导出
    ↓
系统生成文件并触发下载
    ↓
用户完成下载
```

### 6.5 旅程 5：阅读内容并回到方案

```
用户在首页进入「经验心得」
    ↓
用户选择内容条目 → 进入阅读页
    ↓
用户阅读正文
    ↓
用户返回首页
    ↓
用户将阅读中涉及的工具加入方案
    ↓
用户生成并保存方案
```

### 7.3 搜索体系

#### 7.3.1 普通搜索（免费）

**输入**

- `keyword`：用户输入关键词

**搜索范围（默认）**

- 工具：name / description / tags
- 内容：title / summary

**校内专区内容展示规则**

- 未认证用户：普通搜索与 AI 搜索均不得返回任何校内专区内容
- 学生认证用户：校内专区内容在校内专区 Tab 中展示；普通搜索/AI 搜索可按权限返回（仅本校 schoolId 内容）

**输出**

- 工具结果列表（可分页）
- 内容结果列表（可分页）

**排序建议（默认）**

1. 精选内容优先（isFeatured=true）
2. 相关度
3. 热度（views/likes）

#### 7.3.2 AI 搜索（游客可用，本地额度 3 次/天，耗尽降级）

前置约束（V1 演示版）：

- 游客可用；本地额度 3 次/天，localStorage 计数，本机每日 00:00 重置。
- 登录后续版本接入后端鉴权与服务端扣次。

输入：

- `query`：用户自然语言问题
- `domain`：当前领域（creative/dev/work）
- `libraryMode`：'professional' | 'comprehensive'
- `availableTools`：按 domain 与 libraryMode 过滤后的工具列表
- `availableArticles`：按 domain 与 libraryMode 过滤后的内容列表

输出（契约）：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| mode | 'ai' \| 'demo' | 模式标识 |
| fallbackReason | 'missing_key' \| 'network_error' \| 'parse_error' \| 'empty_result' \| 'quota_exhausted' \| '' | 回退原因（枚举类型） |
| summary | string | 中文摘要 |
| recommendation | string | 中文路径建议 |
| suggestedTools | string[] | 推荐工具（≤ 3 个名称） |
| suggestedArticles | string[] | 推荐内容（≤ 2 个标题） |

#### 7.3.3 回退方案（不扣次数）

当 AI 服务不可用/解析失败/输出为空时，进入 demo 回退模式：

- 关键词命中优先（命中工具名优先）
- 其余按评分补齐工具
- 内容优先返回精选

推荐结果区顶部必须展示「演示模式」提示条。

#### 7.3.4 库模式定义

| 模式 | 定义 |
| --- | --- |
| professional（专业库） | 仅使用精选内容与你维护的高质量工具库（例如 isFeatured=true） |
| comprehensive（综合库） | 使用全量工具与内容库（包含广场内容；若用户为学生认证，仅包含其本校校内内容） |

### 7.4 工具选择

#### 7.4.1 选择入口

- 工具卡片提供选择按钮

#### 7.4.2 选择状态

工具选择状态必须**全局持久化**，跨页面保持一致。

#### 7.4.3 浮动 Dock

**出现条件**：已选工具数量 > 0

**Dock 结构**

| 组件     | 说明       |
| ------ | -------- |
| 已选工具预览 | 最多显示 4 个 |
| 已选数量   | 数字显示     |
| 清空选择   | 按钮       |
| 分享     | 按钮       |
| 生成方案   | **主按钮**  |

### 7.5 工具详情

**必须展示内容**

- 工具封面图
- 工具名称
- 评分

## 11. AI 能力规格

#### 11.1 AI 搜索契约

输入字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| query | string | 用户输入字符串 |
| availableTools | Tool[] | 工具数组 |
| availableArticles | Article[] | 内容数组 |
| providerId | string? | 指定厂商（可选） |
| model | string? | 指定模型（可选） |

输出字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| mode | 'ai' \| 'demo' | 模式标识 |
| fallbackReason | string? | 回退原因 |
| summary | string | 摘要 |
| recommendation | string | 路径建议 |
| suggestedTools | string[] | 推荐工具（≤3） |
| suggestedArticles | string[] | 推荐内容（≤2） |

输出必须为 JSON 格式。

#### 展示规则

| 条件          | 展示                  |
| ----------- | ------------------- |
| mode = demo | 在推荐结果区顶部展示「演示模式」提示条 |
| mode = ai   | fallbackReason 为空   |
| mode = demo | fallbackReason 非空   |

### 11.2 失败处理与回退方案

#### 回退触发条件

- AI 不可用
- 解析异常
- 输出为空

#### 回退规则

| 规则      | 说明            |
| ------- | ------------- |
| summary | 必须包含「演示模式」    |
| 工具数量    | 上限 3          |
| 内容数量    | 上限 2          |
| 工具优先级   | 命中关键词优先，按评分补足 |
| 内容优先级   | 精选优先，按列表顺序补足  |

### 11.3 AI 网关与多模型

> **⚠️ V1 实现说明**：V1 仅支持智谱 glm-4.6 单一模型，通过 Vite dev server proxy 转发 AI 请求。以下描述的 AI 网关架构为未来规划，多模型支持计划在 V4 实现。

#### 术语定义

| 术语             | 定义                                       |
| -------------- | ---------------------------------------- |
| AI 网关服务        | 系统内统一的 AI 请求入口服务                         |
| 模型厂商           | zhipu / minimax / qwen / gemini / openai |
| ProviderConfig | 模型服务配置实体                                 |
| Identity       | 用户身份实体，支持匿名                              |
| UsageEvent     | AI 请求用量事件记录                              |

#### AI 能力边界

- 前端应用**不持有**任何第三方模型厂商的真实 API Key
- 所有 AI 请求必须通过 **AI 网关服务**发起
- AI 网关服务返回结果必须遵守前端既有 AI 契约
- 前端允许用户选择模型厂商与模型
- 用户登录后，Identity 绑定到账号

#### AI 网关接口

**GET /api/ai/options**

```
输出：
- providers: Provider[]（id, vendor, models, isDefault, enabled）
- defaultProviderId: string
```

**POST /api/ai/search**

```
输入：query, domain, toolIds, articleIds, providerId, model
输出：AISearchResultV2
```

**POST /api/ai/solution**

```
输入：goal, domain, toolIds, providerId, model
输出：mode, fallbackReason, solutionMarkdown
```

- 移动端（手机浏览器）

#### 18.4.2 代码编写规范

- **兼容性原则**：如无显性兼容性要求，不编写额外的兼容代码（如旧浏览器 polyfill）。

- **代码格式化**：使用 ESLint + Prettier 统一代码风格。

### 18.5 最小化开发原则（已更新）

本 PRD 以实现功能为主，但坚持“少而关键”的交付策略：优先完成搜索与内容资产、AI 次数体系、校内专区与后台配置能力。

- 复杂部署体系与监控平台对接

本 PRD 遵循**最小化开发原则**，以实现功能为主，但允许为长期演进预留合理架构。

**禁止范围**

- 部署方案、CI/CD 流水线
- 监控报警、日志平台对接
- 限流、熔断、服务发现
- 容器化、Kubernetes 编排
- 复杂缓存策略、专项性能工程
- 任何实现功能之外的扩展
