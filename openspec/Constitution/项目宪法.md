# 项目宪法（Constitution）— 盘根 · AI 指南针

版本：v1.0
生效日期：2025-12-22
维护范围：研发 / 设计 / 运营（以本宪法为最高约束）

## 0. 宪法定位
本宪法定义本项目的“不变量（必须长期成立的规则）”。
- 宪法高于任何临时讨论与局部实现。
- 所有功能迭代必须在 `openspec/Changes/` 先形成“图纸”，再进入代码实现。

## 1. 技术栈不变量
### 1.1 前端
前端技术栈固定为：
- 语言：TypeScript
- 框架：React（当前项目使用 React 19）
- 构建：Vite
- 路由：React Router（当前项目使用 v7）
- 样式：Tailwind CSS
- 状态：Zustand（全局轻量状态）
- 表单：React Hook Form + Zod（用于后台表单与创作者中心）
- Markdown 渲染：react-markdown + remark/rehype
- UI 组件：允许逐步引入 shadcn/ui（不强制一次性改造）

### 1.2 兼容性原则
- 如无显性兼容性要求，不编写额外兼容代码（例如旧浏览器 polyfill）。
- 优先使用现代浏览器能力（以桌面 Chrome、移动端 Safari/Chrome 为主）。

### 1.3 后端（未来演进，不阻塞前端原型）
- 允许先以“前端本地存储 + Mock 数据”完成体验闭环。
- 当需要多人协作（作者/审核员/管理员）与成本治理时，引入后端服务与数据库。

## 2. 安全与密钥不变量
- 前端构建产物不得包含任何第三方模型厂商的真实 API Key。
- 开发环境允许使用 Vite dev proxy 转发请求；生产环境必须通过后端 AI 网关服务持有 Key。
- 含密钥的本地配置（例如 `.env.*.local`、`.zhipu.local.json`）必须保持不提交到版本控制。

## 3. 文档与注释语言不变量
- 文档：尽量使用中文编写。
- 代码注释：尽量使用中文编写。
- 变量/函数/类型命名：使用英文驼峰（camelCase / PascalCase）。

## 4. 产品体验不变量（Apple 式克制）
- 最短主路径：任何页面最多 1 个主按钮；主按钮表达“下一步行动”。
- 渐进披露：默认只展示决策所需信息，细节通过抽屉/展开呈现。
- 一致性优先：同类组件在不同页面保持一致（尤其是错误、空态、确认弹窗）。
- 反馈及时：关键操作必须给出 loading/成功/失败/不可撤销提示。
- 禁止：通过“仪式感文案/装饰性动效”制造价值感。

## 5. 多端交付不变量
- 桌面端：Web（浏览器）为主形态。
- 移动端：先把手机网页适配做到“像 App”，再用 WebView 壳（Capacitor）上架 iOS/Android。
- 不维护两套 UI（不采用 React Native 双端独立实现）。
- 移动端交互底线：触控目标 ≥44×44；Safe Area 不遮挡；键盘弹出不挡主操作。

## 6. 功能阶段化上线不变量（Feature Flags）
所有非核心能力必须可阶段化点亮，统一三态策略：
1) Hidden（隐藏）：入口不可见。
2) Locked（锁定态）：入口可见但不可用，点击给出解释与预期。
3) Degraded（降级可用）：功能可用但质量/额度/范围降低，必须有明确提示。

## 7. 内容与文档标准不变量
- 系统内部唯一标准内容格式：Markdown。
- 允许受控扩展块（白名单）：例如 `:::video{...}:::`, `:::callout{...}:::`。
- 禁止任意 HTML 直出（避免安全与样式失控）。

## 8. 后台与协作不变量
- 系统后台是高权限区：仅 reviewer/editor/admin 可访问；普通作者不可见。
- 内容发布必须审核通过（可由授权审核员完成，不强制必须你本人）。
- 发布后允许修改，但必须走二次审核；线上旧版本保持可见，新版本通过后替换。
- 后台 AI 只做“建议/草稿/辅助分析”，禁止自动发布、自动精选、自动下架。
- 指派/改派/AI 操作/版本回滚必须写审计日志。

## 9. 本地存储不变量
- 读写必须有校验（guard），数据损坏自动重置。
- 重置必须提示一次且文案固定：“本地数据异常，系统已重置到默认状态。”
- 写入必须满足原子性（一次 setItem 写入完整 JSON）。

## 10. 参考规格
- 主规格（当前单一事实来源）：`openspec/Specs/PRD-盘根AI指南针-标准版.md`
- 变更规格：`openspec/Changes/` 下最新变更目录
