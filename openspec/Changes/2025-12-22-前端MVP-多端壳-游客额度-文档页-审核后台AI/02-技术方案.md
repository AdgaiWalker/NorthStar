# 技术方案：前端 MVP（多端壳）+ 游客额度 + 文档页 + 审核后台（含 AI 助手）

版本：v0.1
日期：2025-12-22

## 1. 总体架构
### 1.1 目标
- 先完成“前端可演示闭环”，后续再将本地存储能力迁移到后端服务。
- 所有关键能力都具备阶段化开关（Hidden/Locked/Degraded）。

### 1.2 模块划分
- 公共站（Web）：首页、工具详情、文档阅读页、方案生成/保存/导出、我的（基础）。
- 文档系统（Doc）：Markdown 标准 + 受控块（视频/提示块）+ 目录 + 双链。
- 审核后台（Admin）：审核队列、审核详情、待分配池、审核员管理、审计日志。
- 后台 AI：仅在审核详情页提供 AI 助手侧栏（Diff 预览 + 一键应用）。

## 2. 游客 AI 额度（本地存储）
### 2.1 计费桶（分桶）
- ai_search：3 次/天
- ai_solution：3 次/天

### 2.2 重置策略
- 重置时间：本机时间每日 00:00。
- 计数存储：localStorage（早期接受清缓存/换设备会重置）。

### 2.3 额度耗尽的行为
- 不做“不可用硬拦截”。
- 到限额后强制进入 demo 演示模式：
  - 顶部提示条：说明“演示模式（因额度/服务原因）”。
  - 输出结构保持一致（摘要/路径建议/推荐工具/推荐内容）。

### 2.4 本地存储键
在现有 `pangen.*` 前缀下新增：
- `pangen.quota.guest.aiSearch`
- `pangen.quota.guest.aiSolution`
- `pangen.quota.guest.resetAt`（下次重置时间戳）

## 3. 文档页（Markdown + 受控块）
### 3.1 内容标准
- 内部唯一标准：Markdown。
- 图片：仅外链 URL（Markdown `![](https://...)`）。
- 视频：仅支持 `https` 直链 `mp4`（后续再支持平台 iframe）。

### 3.2 受控块语法（白名单）
#### 3.2.1 视频块
- 语法：
  - `:::video{src="https://...mp4" caption="可选说明"}:::`
- 约束：
  - src 必须为 https
  - 文件必须为 mp4
  - 允许一个文档出现多个视频块

#### 3.2.2 提示块（callout）
- 语法：
  - `:::callout{type="tip|warning|info" title="可选标题"} ... :::`
- 用途：将注意事项、坑点、前置条件以 Obsidian 风格提示块呈现。

### 3.3 目录（TOC）生成
- 目录来源：H1–H6 标题。
- 生成策略：
  - 对 Markdown 渲染阶段生成稳定锚点（slug）。
  - 标题重复时自动追加序号，确保锚点唯一。
- 桌面：右侧 TOC 常驻。
- 移动：TOC 入口为按钮，打开抽屉/Sheet。

## 4. 双链（Link）与反向链接（Backlinks）
### 4.1 双链目标类型
- 文档/文章：`[[文档标题]]` 或 `[[文档标题|显示文本]]`
- 工具：`[[tool:工具ID|工具名]]`
- 方案：`[[solution:方案ID|方案标题]]`

### 4.2 插入与联想
- 输入 `[[` 弹出联想列表。
- 默认排序优先文档（符合 Obsidian 习惯）。
- 工具/方案必须插入带类型前缀的链接，避免重名冲突。

### 4.3 未创建处理
- 文档：允许 `[[未存在文档]]`，点击可创建新文档（后续阶段点亮）。
- 工具/方案：不存在时提示“未找到”，不允许自动创建（避免污染库）。

### 4.4 反向链接（后续点亮）
- 当前对象的反向链接 = 所有引用到它的文档集合（含命中片段）。
- 先做结构与数据模型预留，UI 可先 Locked。

## 5. 审核后台（Review Workbench）
### 5.1 核心对象：审核任务 ReviewTask
状态：
- unassigned（待分配池）
- assigned（已分配）
- in_review（已开始审核）
- approved / rejected（已完成）

关键字段：
- taskId, contentId, revisionId?
- domain, contentType
- assignedToUserId?
- submittedAt, assignedAt?, dueAt
- lastActionAt
- unassignedReason?（枚举）

### 5.2 自动分配
- WIP 上限：每个审核员同时最多 10 条（assigned + in_review）。
- SLA：48 小时内完成审核。
- 分配算法：
  - 候选人：接单中 + 可审领域命中 + 未满载。
  - 选择：待审数量最少；并列按“上次分配时间最早”轮转。
- 修订优先同人：同一内容的修订稿优先分配给上次审核该内容的人。

### 5.3 待分配池（严格不超配）
触发原因（unassignedReason 枚举）：
- no_reviewer_available
- all_reviewers_over_capacity
- no_reviewer_for_domain
- policy_blocked

处置原则：
- 自动恢复：当释放容量/有人恢复接单时，尝试重新分配。
- 人工接管：admin 可手动指派或接管。

### 5.4 改派与权限
- reviewer：只审，不改派。
- editor：可改派 assigned 状态任务；不可强制改派 in_review。
- admin：可强制改派 in_review（必须填原因并写审计）。

### 5.5 审计日志（AuditLog）
记录所有关键动作：
- auto_assign / manual_assign / reassign / take_over / auto_reassign_overdue
- reviewer_pause / reviewer_resume
- ai_run / ai_apply / revision_create / revision_rollback

最小字段：
- auditId, timestamp, operatorUserId(system?), action
- taskId, contentId, revisionId?
- fromAssignee, toAssignee
- reasonCode, note?

## 6. 后台 AI 助手（仅审核详情页）
### 6.1 交互原则
- 输出以 Diff 预览呈现。
- 应用必须可撤销，并自动生成修订版本（revision）。
- 默认上下文只读取当前页面内容（page_only）。
- 增强上下文需要管理员显式勾选（enhanced），并显示“本次引用范围与预计成本”。

### 6.2 P0 按钮（审核详情页）
- 快速审读摘要（Summary）
- 拒绝原因草稿（严谨模板）
- 修订稿改动总结（二审）
- 风险提示（只标红命中片段，不自动裁决）

### 6.3 应用策略
- 默认只允许“填入审核表单”（拒绝原因/审核备注）。
- 对内容字段的直接改动仅 editor/admin 可执行，且写入修订稿。

## 7. 端侧适配（移动像 App）
- 桌面：Header + 主内容区 + 右侧 TOC（文档页）
- 移动：底部 Tab；文档 TOC 用抽屉；Dock 采用底部 Sheet（避免挡 Tab 与键盘）。

## 8. 安全与隐私
- 前端不持有真实模型 API Key。
- 后台 AI 同样必须通过 AI 网关（本地开发可沿用 dev proxy）。
- 分享/复制文本不得包含个人隐私字段。
