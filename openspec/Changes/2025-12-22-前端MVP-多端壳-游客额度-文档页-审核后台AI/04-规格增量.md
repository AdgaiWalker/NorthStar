# 规格增量：如何同步到主规格（Specs）

版本：v0.1
日期：2025-12-22

## 1. 本次变更影响的主规格范围
主规格文件：`openspec/Specs/PRD-盘根AI指南针-标准版.md`

本次变更会对以下主题形成增量修订：
- AI 使用规则（游客可用 + 本地额度 + demo 降级）
- 阅读页升级为“文档页”（混排视频块、目录、双链）
- 系统后台从“后续规划”提前落地“审核工作台 + 自动分配 + 审计 + 后台 AI 助手”的最小版本

## 2. 建议新增的 Specs 文件（主规格拆分，避免 PRD 过度膨胀）
建议在 `openspec/Specs/` 新增：
1) `Spec-文档系统-混排块与双链.md`
- Markdown 标准
- 受控块语法（video/callout）
- 目录生成规则
- 双链语法（doc/tool/solution）与未创建处理
- 反向链接（预留）

2) `Spec-审核工作台-任务模型与自动分配.md`
- ReviewTask 状态机
- WIP=10、SLA=48h
- 修订优先同人
- 待分配池原因枚举与处置
- 改派权限（editor 可改派，admin 可强制）

3) `Spec-后台AI助手-交互与审计.md`
- 审核详情页 AI 侧栏
- Diff 预览 + 一键应用
- 默认上下文 page_only
- 应用后自动生成 revision
- 审计字段与成本闸门

## 3. 对 PRD 的具体增量建议（章节级别）
### 3.1 AI 功能“需登录”规则调整
PRD 当前多处描述 AI 功能需登录。本次增量建议改为：
- 游客可用 AI，但有本地额度（3+3/天），额度耗尽进入 demo。
- 登录体系在后续阶段引入，用于：更高额度、多人协作、服务端扣次与成本治理。

### 3.2 阅读页（7.6）升级为文档页
PRD 当前以 `isVideo` 区分视频内容。本次增量建议改为：
- “视频”作为文档内块（可多个），不再是页面级二选一。
- `isVideo` 作为历史字段逐步废弃。

### 3.3 系统后台（9）提前落地最小闭环
PRD 当前系统后台在 V3-V4。本次增量建议：
- 先落地审核工作台最小闭环（队列/详情/改派/审计/待分配池）。
- 后台 AI 助手只做审核提效（拒绝原因草稿、修订总结等），不做自动发布。

## 4. 同步策略
- 本次变更在 `openspec/Changes/.../` 完成并验证后，将以上内容合并到 `openspec/Specs/`。
- 合并时原则：
  - 把“不变量”沉淀到 `openspec/Constitution/项目宪法.md`
  - 把“可变实现细节”沉淀到新增的 Spec 文档
  - PRD 保持“产品视角的最终真相”，避免塞入过多实现细节
