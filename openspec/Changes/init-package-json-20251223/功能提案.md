# 功能提案
背景：当前项目根目录缺少 `package.json`，导致 `pnpm lint` 等脚本无法执行，阻断后续开发与集成。
User Story：作为维护者，我希望在项目根目录拥有规范化的 `package.json`，以便安装依赖、运行脚本并对外声明包元数据。
目标：
- 在根目录生成满足 pnpm 使用的 `package.json`。
- 约定基础脚本（lint 等）与包元信息，保持后续可扩展。
- 保持改动最小化且符合 Node 包命名规范（小写、无空格、无特殊符号）。
非目标：
- 不在本次变更中添加真实 lint 规则或依赖。
- 不发布到 npm registry。
验收标准：
- 根目录存在合法的 `package.json`，字段通过 `pnpm pkg set` 或静态校验无错误。
- 执行 `pnpm lint` 返回成功（即使暂为占位脚本）。
- 包名与版本信息符合命名规范且标记为私有（"private": true）。
