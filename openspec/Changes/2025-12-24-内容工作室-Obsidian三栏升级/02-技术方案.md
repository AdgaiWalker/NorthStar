# 技术方案：内容工作室 Obsidian 三栏升级

## 1. 数据模型扩展

### 1.1 ContentItem 新增字段
```typescript
// src/types.ts 新增
interface ContentItem {
  // ... 现有字段
  folder?: string;  // 虚拟文件夹路径，例如 "/创作/AI教程"
}
```

### 1.2 FolderMeta 新类型
```typescript
// src/types.ts 新增
interface FolderMeta {
  path: string;           // 文件夹路径，例如 "/创作"
  domain?: Domain;        // 绑定的领域（可选，用于继承）
  createdAt: string;      // ISO 时间
}
```

### 1.3 存储键
- `pangen.content.folders`: `FolderMeta[]`

## 2. Store 扩展：useContentStore

### 2.1 新增 State
```typescript
folders: FolderMeta[];
selectedFolder: string | null;  // 当前选中的文件夹路径
```

### 2.2 新增 Actions
```typescript
createFolder: (path: string, domain?: Domain) => void;
updateFolder: (path: string, patch: { domain?: Domain }) => void;
deleteFolder: (path: string) => void;
setSelectedFolder: (path: string | null) => void;
moveContent: (contentId: string, targetFolder: string) => void;
```

### 2.3 新增 Selectors
```typescript
getContentsByFolder: (folder: string | null) => ContentItem[];
getFolderTree: () => TreeNode[];  // 返回嵌套树结构
getInheritedDomain: (folder: string) => Domain | undefined;
```

## 3. UI 布局调整

### 3.1 整体三栏布局（桌面端）
```
┌──────────────────────────────────────────────────────────────────┐
│  Header（顶部工具栏：返回/保存状态/发布按钮）                          │
├────────────┬─────────────────────────────┬──────────────────────┤
│  文件树     │  编辑器（CodeMirror）         │  属性检视器            │
│  240px     │  flex-1                      │  280px               │
│            │                              │                      │
│  📁 创作    │  [Markdown Editor]          │  ▼ 基础属性            │
│    📄 a1   │                              │    标题：xxx          │
│    📄 a2   │                              │    文件夹：/创作       │
│  📁 开发    │                              │    领域：继承(创作)    │
│  📁 办公    │                              │  ▼ 发布设置            │
│            │                              │    可见性：public     │
│  ──────    │                              │    学校ID：           │
│  📄 未分类   │                              │  ▸ 高级属性（折叠）     │
└────────────┴─────────────────────────────┴──────────────────────┘
```

### 3.2 响应式策略
- **≥1280px**：三栏全部展示
- **≥768px**：文件树收起为图标，点击展开抽屉
- **<768px**：文件树和属性面板均为底部抽屉

## 4. 组件设计

### 4.1 FileTree 组件
文件：`src/components/admin/FileTree.tsx`
Props：
- `folders: FolderMeta[]`
- `contents: ContentItem[]`
- `selectedFolder: string | null`
- `selectedContentId: string | null`
- `onSelectFolder: (path: string | null) => void`
- `onSelectContent: (id: string) => void`
- `onCreateFolder: (path: string) => void`
- `onDeleteFolder: (path: string) => void`

行为：
- 点击文件夹：展开/折叠子节点，并设置 selectedFolder
- 点击文件（ContentItem）：跳转到编辑页
- 右键菜单：新建文件夹/删除文件夹/设置领域
- 拖拽：将内容拖入文件夹（调用 moveContent）

### 4.2 PropertiesPanel 组件
文件：`src/components/admin/PropertiesPanel.tsx`
Props：
- `item: ContentItem`
- `inheritedDomain: Domain | undefined`
- `onUpdate: (patch: Partial<ContentItem>) => void`

结构：
```jsx
<CollapsibleSection title="基础属性" defaultOpen>
  <TextField label="标题" value={item.title} />
  <FolderSelect value={item.folder} />
  <DomainBadge inherited={!!inheritedDomain} value={item.domain} />
</CollapsibleSection>
<CollapsibleSection title="发布设置" defaultOpen>
  <VisibilitySelect value={item.visibility} />
  <SchoolIdInput show={item.visibility === 'campus'} />
</CollapsibleSection>
<CollapsibleSection title="高级属性" defaultOpen={false}>
  <TagsInput value={item.tags} />
  <RelatedToolsSelect value={item.relatedToolIds} />
  <CoverImageInput value={item.coverImageUrl} />
</CollapsibleSection>
```

### 4.3 ContentStudio 页面
文件：`src/pages/admin/ContentStudioPage.tsx`

整合 FileTree、MarkdownEditor、PropertiesPanel，使用 ResizablePanelGroup 实现可调宽度。

## 5. 路由调整
- `/admin/content` → 内容列表（保留，用于批量管理）
- `/admin/studio` → 内容工作室（新增，Obsidian 风格）
- `/admin/studio/:contentId` → 直接打开指定内容

## 6. API 契约（本地实现）
无后端依赖，所有数据存储于 localStorage。
