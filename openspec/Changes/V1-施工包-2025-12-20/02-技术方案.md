# 02-技术方案 (Technical Specification)

## 1. 架构设计

### 1.1 系统边界
- **V1 架构**：纯前端应用（Client-Side App），数据持久化依赖 `localStorage`。
- **AI 交互**：前端通过 Next.js API Route 或开发环境 Proxy 转发请求，严禁 Key 暴露于前端构建产物。

### 1.2 模块划分
- **Domain Layer**：`types.ts` 定义核心实体（Tool, Article, UserSolution）。
- **Service Layer**：`AIService.ts` 封装 AI 请求与回退逻辑；`storage.ts` 封装本地存储读写。
- **UI Layer**：`components/` 复用组件（Card, Layout）；`App.tsx` 页面路由与状态组装。

## 2. 数据契约

### 2.1 本地存储 (LocalStorage)
| Key | Type | Description |
| :--- | :--- | :--- |
| `pangen.themeMode` | `light \| eye-care` | 主题偏好 |
| `pangen.language` | `zh \| en \| jp \| ru` | 语言偏好 |
| `pangen.selectedToolIds` | `string[]` | 已选工具 ID 集合 |
| `pangen.userSolutions` | `UserSolution[]` | 用户生成的方案列表 |

**一致性策略**：启动时读取，写入时全量覆盖（原子性），损坏时重置并提示。

### 2.2 AI 搜索契约 (V1)
- **输入**：`query` (string), `availableTools` (Tool[]), `availableArticles` (Article[])
- **输出** (JSON):
  ```typescript
  interface AISearchResult {
    mode: 'ai' | 'demo';
    fallbackReason?: 'missing_key' | 'network_error' | 'parse_error' | 'empty_result';
    summary: string; // 必须包含中文摘要
    recommendation: string; // 路径建议
    suggestedTools: string[]; // 工具名称数组（Max 3）
    suggestedArticles: string[]; // 文章标题数组（Max 2）
  }
  ```
- **回退逻辑**：
  - 当 API 失败或 Key 缺失时，`mode` 强制设为 `demo`。
  - Demo 模式下，基于关键词匹配与评分（Rating）生成静态推荐结果。
  - Demo 模式必须在 UI 展示黄色警告条。

## 3. 关键交互实现

### 3.1 浮动 Dock
- **触发条件**：`selectedToolIds.length > 0`
- **状态同步**：Dock 操作（清空、删除）必须实时同步至 LocalStorage。

### 3.2 方案生成
- **Mock 延迟**：模拟 1.5s 生成过程，展示 Loading 动画与随机提示语。
- **数据组装**：结合“已选工具”与“用户目标”，通过模板生成 Markdown 格式的 `aiAdvice`。

## 4. 安全约束
- **API Key**：仅在 `.env` 或后端环境配置，前端代码中不得出现硬编码 Key。
