# V2 变更包（盘根 · AI 指南针）技术方案：可维护性增强

## 1. 技术栈与工程形态（不变项）

本变更包不改变既有技术栈：

- 前端框架：React 19
- 语言：TypeScript
- 构建工具：Vite
- 样式系统：TailwindCSS
- Markdown 渲染：react-markdown
- AI SDK：@google/genai
- 包管理：npm

## 2. 目标架构原则

- **P1 单一事实来源**：规格与宪法仅保留一套根目录 `openspec/`。
- **P2 显式契约**：跨模块交互全部通过类型与字段表达，不使用隐式文案规则做逻辑判断。
- **P3 横切能力下沉**：存储、过滤、AI 回退属于横切能力，放入 `1pangen-ai-compass/services/` 与 `1pangen-ai-compass/utils/`。
- **P4 入口文件轻量化**：`App.tsx` 只负责装配、注入、顶层视图切换。

## 3. 模块拆分设计

### 3.1 新增目录

在 `1pangen-ai-compass/` 内新增：

- `utils/`
  - `storage.ts`：本地存储读写与校验
  - `guards.ts`：类型守卫与数据校验
- `services/`
  - `aiFallback.ts`：回退结果生成器
  - `aiContract.ts`：AI 搜索输出类型与模式字段

说明：`services/geminiService.ts` 保留为 AI 调用适配层，回退策略抽离到 `services/aiFallback.ts`。

### 3.2 本地存储封装

#### 3.2.1 键名集中

在 `utils/storage.ts` 内定义：

- `STORAGE_KEYS.themeMode`
- `STORAGE_KEYS.language`
- `STORAGE_KEYS.selectedToolIds`
- `STORAGE_KEYS.userSolutions`

键名取值必须与主 PRD 第 13 章一致。

#### 3.2.2 读取与写入 API

在 `utils/storage.ts` 内提供明确函数：

- `loadThemeMode(): ThemeMode`
- `saveThemeMode(value: ThemeMode): void`
- `loadLanguage(): Language`
- `saveLanguage(value: Language): void`
- `loadSelectedToolIds(): Set<string>`
- `saveSelectedToolIds(value: Set<string>): void`
- `loadUserSolutions(): UserSolution[]`
- `saveUserSolutions(value: UserSolution[]): void`

读取行为：

- 初次读取在 `useState` 惰性初始化阶段执行
- 解析失败时删除对应键名，并返回默认值
- 解析失败时必须向上返回 `storageResetDetected = true` 的信号，由 UI 在启动阶段提示一次

写入行为：

- 在 `useEffect` 监听状态变更写入
- 写入必须使用单次 `setItem` 完成

#### 3.2.3 校验策略

- JSON 解析使用 `safeJsonParse`
- 数组与对象使用类型守卫校验
- `guards.ts` 只包含纯函数，不依赖 React

### 3.3 AI 契约显式化

#### 3.3.1 新类型

在 `services/aiContract.ts` 定义：

- `AISearchMode`：`ai`、`demo`
- `AISearchFallbackReason`：`missing_key`、`network_error`、`parse_error`、`empty_result`
- `AISearchResultV2`：在现有字段基础上新增：
  - `mode: AISearchMode`
  - `fallbackReason: AISearchFallbackReason | null`

兼容要求：

- 主 PRD 已规定回退结果 `summary` 必须包含“演示模式”，该规则继续满足
- UI 判断演示模式使用 `mode === 'demo'`

#### 3.3.2 回退生成器

在 `services/aiFallback.ts` 定义 `buildFallbackResult`，输入：

- `query: string`
- `availableTools: Tool[]`
- `availableArticles: Article[]`
- `reason: AISearchFallbackReason`

输出：`AISearchResultV2`

排序规则：

- 工具：命中关键词优先，未命中部分按 `rating` 从高到低补足
- 内容：`isFeatured` 为 true 的内容优先，其余按原列表顺序补足
- 数量：工具上限 3，内容上限 2

### 3.4 领域过滤边界

在 `App.tsx` 层完成领域过滤并传入 `AISearch`：

- `tools` 仅传入当前领域工具
- `articles` 仅传入当前领域内容

该规则保持不变，后续将过滤函数抽离到 `utils/domainFilter.ts`。
