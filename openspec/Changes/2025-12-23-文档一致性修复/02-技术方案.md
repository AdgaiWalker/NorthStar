## 2. 修复策略（以代码为真，合同优先）
1) 安全（P0）：为 `/admin` 增加最小路由守卫  
   - 方案：使用前端模拟登录状态 `isLoggedIn` + 角色占位（admin），未登录跳转 `/login`。保持无后端前提下的最小防护。
2) 契约与配置（P1）  
   - fallbackReason 增补 `quota_exhausted` 到 PRD；README 指向正确 PRD；README 说明 model 固定或支持读取 env；额度规则文档改为“每日 00:00”以对齐实现。  
   - README 增补 build/preview/lint/format 命令与 lockfile 策略说明。
3) 逻辑小修（P1/P2）  
   - 普通搜索匹配 tags；方案生成失败展示错误/重试提示，不再直接跳转；文案与本地额度策略一致。  
   - 阅读页/文档系统在规格中声明受控块（video/callout）与双链行为；PRD isVideo 段落改为“视频为块级受控语法”。  
   - User/Review 契约在 PRD 中降级为当前实现字段，并在 Specs 标注“V1 Demo 简化”。
4) 规格同步  
   - 在 `openspec/Specs/` 增补三份分拆 Spec（文档系统、审核工作台、后台 AI 助手）或在本次变更内以“待创建”方式记录并引用。

## 3. 变更清单（按文件）
- README：修正 PRD 链接；增加脚本/lockfile/模型配置说明；澄清登录门槛与额度规则。
