## 1. 背景与目标
上一轮审计发现 README、主规格（PRD）与前端实现存在 31 处不一致，含 1 个 P0（后台无访问控制）、9 个 P1、21 个 P2。需在不引入后端的前提下，优先修复安全与契约类问题，并同步文档。

## 2. 范围（本次必须完成）
- 安全：为 `/admin` 加最小访问控制（登录/角色占位），消除 P0。
- 契约与文档：对齐 README 与 PRD 中的关键承诺（AI 登录门槛、额度规则、fallback 枚举、文件路径）。
- 配置与可复现：补充缺失的安装/构建说明、lockfile 策略说明。
- UI/逻辑小修：普通搜索包含 tags；方案生成流程文案/提示与实现一致；阅读页文档块能力在规格中声明。

## 3. 非目标
- 不实现真实后端认证/扣次；仍保持前端本地演示模式。
- 不上线支付/学生认证/投稿等 V2+ 功能，仅做文档降级或 Locked 提示。

## 4. 验收标准
- P0 清零；P1、P2 完成文档与代码双向收敛，残留列入“待修复”清单。
- README、PRD、变更文档行文一致，链接可访问，可执行命令与 package 脚本一致。
- 新增/修改点均有行号级证据，执行清单全部勾选。
