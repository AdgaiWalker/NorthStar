# V1 施工包（盘根 · AI 指南针）功能提案

## 1. User Story

### 1.1 作为新手执行者

我在首页输入真实问题后，系统在 10 秒内输出一条可执行路径，并引导我把路径保存为方案，从而让我立即开始推进任务。

### 1.2 作为进阶整合者

我在浏览工具时把多个工具加入已选集合后，系统给出一条分步骤方案，并允许我导出为文件，从而让我把工作流沉淀为可复用资产。

### 1.3 作为内容学习者

我在阅读经验内容后，能够把内容中涉及的工具快速加入已选集合，再生成方案，从而让我把学习转化为行动。

## 2. 目标（Goals）

- 目标 1：交付“问题 → 推荐 → 选工具 → 生成方案 → 保存 → 导出”的主路径闭环。
- 目标 2：交付“AI 不可用 → 回退方案 → 非空推荐”的体验兜底闭环。
- 目标 3：交付“我的方案”资产管理闭环，包含列表、详情、删除、导出。

## 3. 非目标（Non-goals）

- 不交付真实支付闭环。
- 不交付真实投稿审核闭环。
- 不交付完整账号权限体系。
- 不交付多模型后台配置界面。

## 4. 约束（Constraints）

- 约束 1：严格遵守 `openspec/Specs/PRD-盘根AI指南针-最终版.md` 的需求与字段定义。
- 约束 2：每个原子任务改动控制在 50 行以内，单次提交只实现一个可验证的行为变化。
- 约束 3：前端不写入任何密钥，不在 UI 中展示任何密钥相关信息。

## 5. 交付物（Deliverables）

- 交付物 1：V1 主路径闭环在现有 React + TypeScript 前端工程中可用。
- 交付物 2：本地存储写入与读取满足 PRD 定义的字段与一致性要求。
- 交付物 3：AI 搜索请求与回退方案行为满足 PRD 的输入输出契约。
