# V1 施工包（盘根 · AI 指南针）技术方案

## 1. 技术栈与工程形态

### 1.1 技术栈

- 前端框架：React 19
- 语言：TypeScript
- 构建工具：Vite
- 样式系统：TailwindCSS（通过 `index.html` 注入）
- Markdown 渲染：react-markdown
- AI SDK：@google/genai

### 1.2 工程模块边界

- `1pangen-ai-compass/App.tsx`
  - 负责全局状态
  - 负责视图切换
  - 负责工具选择集合
  - 负责方案生成视图
  - 负责方案保存、删除、导出
  - 负责 Dock 分享
- `1pangen-ai-compass/components/AISearch.tsx`
  - 负责搜索输入
  - 负责 AI 请求触发
  - 负责结果区展开与折叠
  - 负责推荐工具、推荐内容的映射与展示
- `1pangen-ai-compass/services/geminiService.ts`
  - 负责 AI 请求封装
  - 负责 JSON 响应解析
  - 负责失败兜底的回退方案输出
- `1pangen-ai-compass/types.ts`
  - 负责全量数据类型定义
- `1pangen-ai-compass/constants.ts`
  - 负责工具、内容、主题的模拟数据

## 2. 数据结构（Data Schema）

本节定义本次 V1 施工直接依赖的数据结构事实，字段必须与 `types.ts` 保持一致。

### 2.1 领域（Domain）

- 领域代码：`creative`、`dev`、`work`
- 领域展示名：影视创作、编程开发、通用办公
- 领域用途：
  - 工具筛选
  - 内容筛选
  - AI 搜索上下文收敛

### 2.2 工具（Tool）

字段定义：

- `id`：字符串
- `name`：字符串
- `description`：字符串
- `fullDescription`：字符串
- `domain`：Domain
- `tags`：字符串数组
- `rating`：数字
- `usageCount`：字符串
- `imageUrl`：字符串
- `url`：字符串
- `isFavorite`：布尔

### 2.3 内容（Article）

字段定义：

- `id`：字符串
- `topicId`：字符串，允许为空
- `title`：字符串
- `summary`：字符串
- `content`：字符串，Markdown
- `domain`：Domain
- `author`：字符串
- `authorLevel`：字符串，取值 `certified`、`user`
- `date`：字符串
- `readTime`：字符串
- `relatedToolId`：字符串，允许为空
- `imageUrl`：字符串
- `isVideo`：布尔
- `isFeatured`：布尔
- `stats.views`：数字
- `stats.likes`：数字
- `stats.comments`：数字

### 2.4 方案（UserSolution）

字段定义：

- `id`：字符串
- `title`：字符串
- `targetGoal`：字符串
- `toolIds`：字符串数组
- `aiAdvice`：字符串，Markdown
- `createdAt`：字符串

## 3. 本地存储方案（Local Storage）

### 3.1 存储范围

V1 必须持久化以下状态：

- 已选工具集合
- 我的方案列表
- 主题偏好
- 语言偏好

### 3.2 键名规范

- `pangen.themeMode`
- `pangen.language`
- `pangen.selectedToolIds`
- `pangen.userSolutions`

### 3.3 序列化格式

- `pangen.themeMode`
  - 取值集合：`light`、`eye-care`
  - 存储类型：字符串
- `pangen.language`
  - 取值集合：`zh`、`en`、`jp`、`ru`
  - 存储类型：字符串
- `pangen.selectedToolIds`
  - 存储类型：JSON 字符串
  - JSON 结构：字符串数组
  - 数组元素：工具 `id`
- `pangen.userSolutions`
  - 存储类型：JSON 字符串
  - JSON 结构：`UserSolution` 数组

### 3.4 读写时机

- 读取时机：应用首帧渲染前的状态初始化阶段
- 写入时机：状态变更后立即写入
- 写入原则：单键单次写入，避免跨键写入导致中间态

### 3.5 数据损坏处理

- 损坏判定：
  - JSON 解析抛出异常
  - JSON 根结构不是数组
  - 数组元素类型不符合字段定义
- 处理动作：
  - 回到默认值
  - 清理对应键名
  - 展示一次提示文案
- 提示文案：本地数据异常，系统已重置到默认状态。

## 4. AI 搜索契约实现

### 4.1 输入契约

AI 搜索调用必须提供：

- `query`：用户输入字符串
- `availableTools`：当前领域下的工具数组
- `availableArticles`：当前领域下的内容数组

### 4.2 输出契约

AI 搜索输出必须返回结构化对象：

- `summary`：字符串
- `recommendation`：字符串
- `suggestedTools`：字符串数组
- `suggestedArticles`：字符串数组

输出必须可被 JSON 解析。

### 4.3 回退方案输出

回退方案在以下条件触发：

- 未注入密钥导致 AI 调用不可用
- 网络异常导致 AI 调用失败
- JSON 解析失败
- 输出列表为空

回退方案输出规则：

- 工具候选：
  - 规则 1：当 `query` 命中工具名称时，将该工具置顶
  - 规则 2：按 `rating` 从高到低补足，数量上限 3
- 内容候选：
  - 规则 1：优先选择 `isFeatured=true` 的内容
  - 规则 2：按列表顺序补足，数量上限 2
- 文案要求：
  - `summary` 必须包含“演示模式”
  - 推荐结果区必须展示“演示模式”提示语

## 5. 导出与分享实现

### 5.1 导出

- 导出入口：我的方案列表条目按钮、方案详情抽屉按钮
- 支持格式：Markdown、TXT、CSV
- 文件名规则：将标题空格替换为下划线

### 5.2 分享

- Dock 分享：复制工具清单文本
- 方案分享：复制方案摘要文本
