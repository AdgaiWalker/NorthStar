# 技术方案：用户中心（收藏/设置/额度/认证）+ 校内专区（锁定态与访问控制）

版本：v0.1
日期：2025-12-23

## 1. 总体实现策略
系统以“前端可演示闭环”为目标，严格遵守项目宪法：
- 所有新增状态进入 `useAppStore`，并通过 `utils/storage.ts` 的 key + guard 做持久化与校验。
- 所有入口遵守三态策略（Hidden/Locked/Degraded）；本次主要新增 Locked（校内专区/认证相关）。
- 不引入后端依赖；不引入敏感密钥；不引入兼容性冗余代码。

## 3. 数据模型（Data Schema）
### 3.1 新增本地存储键（STORAGE_KEYS）
在 `pangen-ai-compass/src/utils/storage.ts` 中新增：
- `pangen.favoriteToolIds`：收藏工具 ID 列表（string[]）
- `pangen.export.defaultFormat`：默认导出格式（'md'|'txt'|'csv'）
- `pangen.user.studentCertification`：学生认证状态对象（StudentCertification）

### 3.2 类型定义（TypeScript）
在 `pangen-ai-compass/src/types.ts` 中新增：
- `export type ExportFormat = 'md' | 'txt' | 'csv'`
- `export type CertificationStatus = 'none' | 'pending' | 'verified' | 'rejected'`
- `export interface StudentCertification { status: CertificationStatus; schoolId?: string; schoolName?: string; submittedAt?: string; reviewedAt?: string; rejectReason?: string }`

### 3.3 Guard（数据校验）
在 `pangen-ai-compass/src/utils/guards.ts` 中新增：
- `isExportFormat(value): value is ExportFormat`
- `isStudentCertification(value): value is StudentCertification`

系统在读取本地存储时严格执行：
- 校验通过：使用存储值。
- 校验失败：删除损坏值并回退默认值，同时触发一次性 reset 提示（沿用 App 现有提示机制）。

## 11. UI 交互与文案标准
- Locked 卡片统一结构：
  - 标题（说明当前状态）
  - 一句话解释（为什么不可用）
  - 主按钮（下一步行动）
- 空态统一结构：
  - 图标/标题/解释/主按钮
- 不使用 emoji 作为关键表达，使用图标与排版表达状态。
- 所有新增代码注释使用中文，命名使用英文驼峰。
