# 规格增量：如何同步到主规格（Specs）

版本：v0.1
日期：2025-12-23

## 1. 本次变更影响的主规格范围
主规格文件：`openspec/Specs/PRD-盘根AI指南针-标准版.md`

本次变更将对以下主题形成增量修订：
- 用户后台（8）：从“仅基础 UI”升级为“收藏/设置/额度/学生认证”的可演示闭环。
- 校内专区（4/5/7）：补齐校内专区入口、锁定态策略、访问控制与验收口径。
- 本地存储规范（10.17）：新增收藏/导出偏好/认证状态的存储键与 guard 约束。
- 验收标准（16.4）：补齐“收藏/设置/我的额度/学生认证/校内专区”的可验收项。

## 2. 建议新增的 Specs 文件（主规格拆分）
为避免 PRD 过度膨胀，本次变更将新增两份分册 Spec：

1) `openspec/Specs/Spec-用户中心-收藏设置-额度与认证.md`
- 用户中心信息架构（tabs、入口）
- 收藏的数据模型与交互（收藏/取消、加入方案）
- 设置项（主题/语言/默认导出格式）与“即时生效”策略
- 我的额度页（展示口径、重置时间、demo 不扣次解释）
- 学生认证演示状态机（none/pending/rejected/verified）

2) `openspec/Specs/Spec-校内专区-访问控制与演示数据.md`
- 校内内容标记字段（visibility/schoolId）
- 列表过滤规则（公开/校内）
- 详情页访问控制拦截（直链）
- Locked/Available 的 UI 规范与文案
- 演示数据策略（内置学校 heihe 与校内内容样例）

## 3. 对 PRD 的章节级增量建议
### 3.1 更新 V1 实现范围说明（4.1）
系统将补充 V1 已落地能力：
- 我的：收藏夹、设置、我的额度、学生认证（前端演示）
- 校内专区：入口 + 锁定态 + 访问控制（前端演示）

### 3.2 更新“用户后台（8）V1 实现说明”
系统将把“仅基础 UI”调整为“可演示闭环”，并明确后续版本仍需后端接入（服务端额度/真实认证/运营后台配置）。

### 3.3 更新验收标准（16.4）
系统将新增或细化验收项：
- 收藏：可收藏/取消、收藏页可见、可加入方案
- 设置：主题/语言即时生效，默认导出格式影响快捷导出
- 我的额度：展示与真实扣次规则一致，重置时间正确
- 学生认证：提交→审核中→通过后解锁校内专区
- 校内访问控制：无权限直链访问被拦截并引导认证

## 4. 同步策略
- 本次变更在 `openspec/Changes/.../` 完成并人工验收后，将新增分册 Spec 写入 `openspec/Specs/`。
- PRD 保持产品视角的最终真相，避免塞入过多实现细节；实现细节沉淀到分册 Spec。
- `openspec/Specs/主规格.md` 将更新索引，确保 PRD 与分册 Spec 可被快速定位。
