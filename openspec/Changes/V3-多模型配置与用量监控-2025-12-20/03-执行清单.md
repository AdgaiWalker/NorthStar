# V3 变更包（盘根 · AI 指南针）执行清单：多模型配置与用量监控

> 说明：本执行清单用于未来施工拆解。本次仅产出图纸，不进入施工。

## 任务列表

### [ ] T1 宪法增量：新增服务端技术栈约束

- **改动范围**：`openspec/Constitution/TechStack.md`
- **验收标准**：
  - 文档明确后端框架、数据库、ORM、鉴权、加密策略
  - 文档不包含任何真实 Key、真实地址

### [ ] T2 数据库 Schema 定义

- **改动范围**：后端数据库迁移文件
- **验收标准**：
  - ProviderConfig、ModelConfig、User、UsageEvent、AuditLog 五类表结构完整
  - ProviderConfig 表中仅存储加密 Key

### [ ] T3 后端项目骨架

- **改动范围**：`server/` 目录、基础路由、基础中间件
- **验收标准**：
  - 启动后端服务成功
  - 具备健康检查接口

### [ ] T4 鉴权与管理员权限

- **改动范围**：用户登录接口、JWT 校验中间件、角色校验、Identity 绑定逻辑
- **验收标准**：
  - 未登录访问管理后台接口返回拒绝
  - 非管理员访问管理后台接口返回拒绝
  - 登录成功后服务端将当前 Identity 绑定到账号
  - 未携带 Identity 访问运行时 AI 网关接口时服务端生成 Identity 并下发 cookie

### [ ] T5 Key 加密服务

- **改动范围**：加密模块、主密钥注入机制
- **验收标准**：
  - 写入数据库前完成加密
  - 读取使用前完成解密
  - 后端日志不出现 Key 明文

### [ ] T6 Provider Registry 与 Adapter 接口

- **改动范围**：ProviderAdapter 接口、Registry 注册机制
- **验收标准**：
  - 业务层可以通过 vendor 动态选择 Adapter
  - 新增 vendor 不修改业务 UI 代码

### [ ] T7 首批厂商 Adapter 实现

- **改动范围**：`zhipu`、`minimax`、`qwen`、`gemini`、`openai` 五个 Adapter
- **验收标准**：
  - 每个 Adapter 支持 baseUrl + apiKey + model 调用
  - 统一输出 `{ text, inputTokens, outputTokens }`

### [ ] T8 管理后台接口：ProviderConfig CRUD

- **改动范围**：`/admin/ai/providers` 相关接口
- **验收标准**：
  - 新增、编辑、启用、停用、删除、设置默认接口可用
  - 返回值包含 `apiKeyMask`，不包含密文与明文

### [ ] T9 管理后台接口：连通性测试、轮换 Key、审计日志

- **改动范围**：`/admin/ai/providers/{id}/test`、`/rotate-key`、AuditLog 写入
- **验收标准**：
  - 测试接口返回耗时与失败原因
  - 轮换 Key 后旧 Key 立即失效
  - 所有变更写入审计日志

### [ ] T10 运行时接口：AI 网关

- **改动范围**：`/api/ai/options`、`/api/ai/search`、`/api/ai/solution`
- **验收标准**：
  - `GET /api/ai/options` 返回启用的 Provider 与模型清单
  - AI 请求支持 `providerId` 与 `model` 字段
  - 当 `providerId` 与 `model` 均为空时使用 Identity 首选配置，再使用全站默认 ProviderConfig
  - 当 `providerId` 与 `model` 均为非空时写入 Identity 首选配置
  - 返回结果符合既有前端契约
  - 失败时返回演示模式结果并带 `fallbackReason`

### [ ] T11 用量事件记录

- **改动范围**：UsageEvent 写入逻辑
- **验收标准**：
  - 每次 AI 请求必写 UsageEvent
  - 事件包含 identityId、userId、厂商、模型、耗时、状态、Tokens、成本估算

### [ ] T12 监控查询接口

- **改动范围**：`/admin/ai/usage/overview`、`/events`、`/export`
- **验收标准**：
  - 支持时间范围查询
  - 支持按用户、Identity、厂商、模型、功能模块筛选
  - 导出 CSV 内容完整

### [ ] T13 前端改造：AI 调用切换到网关

- **改动范围**：前端 AI Service 层
- **验收标准**：
  - 前端不再持有真实 Key
  - 前端请求只发往网关
  - 前端通过 `GET /api/ai/options` 获取可选厂商与模型清单
  - 前端 AI 请求携带用户选择的 `providerId` 与 `model`

### [ ] T14 前端改造：后台管理 UI

- **改动范围**：AdminView、Provider 管理页、用量监控页、审计日志页
- **验收标准**：
  - 支持配置新增、编辑、测试、轮换
  - 支持监控指标展示、筛选、导出，筛选包含 userId 与 identityId

### [ ] T15 规格回写归档

- **改动范围**：主 PRD、宪法 TechStack
- **验收标准**：
  - 将本变更包规格增量合并进主规格
  - 完整记录变更范围与约束
