# V3 变更包（盘根 · AI 指南针）规格增量：多模型配置与用量监控

## 1. 本次增量范围

本次规格增量覆盖以下能力：

- 多模型厂商接入机制（配置驱动 + Adapter 接口）
- 模型服务配置后台管理（厂商、Base URL、API Key、默认模型、启用状态）
- 用量监控（按用户、厂商、模型、功能模块统计与明细查询）
- 安全与审计（Key 加密存储、脱敏展示、权限校验、审计日志）
- 匿名身份绑定与用户模型选择

## 2. 主规格增量（写入 openspec/Specs/PRD-盘根AI指南针-最终版.md）

### 2.1 新增术语与概念

主 PRD 必须新增以下术语定义：

- `AI 网关服务`：系统内统一的 AI 请求入口服务，负责路由到指定厂商并返回统一契约。
- `模型厂商 (vendor)`：模型服务提供方标识，取值集合为：`zhipu`、`minimax`、`qwen`、`gemini`、`openai`。
- `模型 (model)`：厂商下的具体模型名称。
- `ProviderConfig`：模型服务配置实体，包含厂商、Base URL、Key、默认模型、启用状态、可用模型列表。
- `Identity`：用户身份实体，支持匿名身份与账号绑定，承载首选厂商与首选模型。
- `UsageEvent`：一次 AI 请求的用量事件记录。

### 2.2 AI 能力边界调整

主 PRD 必须新增明确约束：

- 前端应用不持有任何第三方模型厂商的真实 API Key。
- 所有 AI 请求必须通过 AI 网关服务发起。
- AI 网关服务对外返回结果必须遵守前端既有 AI 契约。
- 前端允许用户选择模型厂商与模型，服务端按该选择发起请求。
- 用户登录成功后，服务端必须将当前 Identity 绑定到账号，历史用量按 Identity 聚合到账号维度展示。

### 2.3 AI 网关接口契约新增

主 PRD 必须新增 AI 网关接口定义：

- `GET /api/ai/options`
  - 输出字段：启用的 Provider 与模型清单、全站默认 Provider

- `POST /api/ai/search`
  - 输入字段：`query`、`domain`、`toolIds`、`articleIds`、`providerId`、`model`
  - 输出字段：`AISearchResultV2`

- `POST /api/ai/solution`
  - 输入字段：`goal`、`domain`、`toolIds`、`providerId`、`model`
  - 输出字段：`mode`、`fallbackReason`、`solutionMarkdown`

### 2.4 管理后台信息架构新增

主 PRD 必须新增管理后台信息架构：

- 一级导航：`AI 引擎`
- 二级页面：
  - `模型服务配置`
  - `用量监控`
  - `审计日志`

### 2.5 模型服务配置页规格新增

主 PRD 必须新增模型服务配置页规格，包含：

- 列表字段：展示名、厂商、Base URL、默认模型、启用状态、是否默认、Key 状态（脱敏）、更新时间
- 操作能力：新增、编辑、启用、停用、设置默认、连通性测试、轮换 Key、删除
- 交互约束：
  - Key 在界面中必须遮罩展示
  - Key 仅在创建与轮换时输入
  - 配置保存成功后立即生效

### 2.6 用量监控页规格新增

主 PRD 必须新增用量监控页规格，包含：

- 筛选条件：时间范围、厂商、模型、用户、Identity、功能模块
- 概览指标：请求次数、成功次数、失败次数、成功率、平均延迟、Tokens 总量、成本估算
- 趋势统计：按天聚合的请求次数、Tokens、成本
- 明细列表字段：时间、用户、厂商、模型、功能模块、耗时、状态、错误类别、Tokens、成本
- 导出能力：导出当前筛选条件下的明细 CSV

### 2.7 安全与审计规格新增

主 PRD 必须新增安全与审计规则：

- API Key 必须加密存储于服务端数据库。
- 任何 API 响应禁止返回 Key 明文与 Key 密文。
- 任何日志禁止记录 Key 明文与 Key 密文。
- 管理后台接口必须进行管理员权限校验。
- 模型配置的新增、编辑、删除、启用、停用、设置默认、轮换 Key 必须写入审计日志。

## 3. 宪法增量（写入 openspec/Constitution/TechStack.md）

TechStack 宪法必须新增后端技术栈条目（作为目标约束）：

- Node.js（TypeScript）
- Fastify
- PostgreSQL
- Prisma
- JWT
- AES-256-GCM

说明：本次仅写入宪法约束，不进入代码施工。

## 4. 影响面说明

- 前端侧：AI Service 调用路径从直连厂商 SDK 调整为调用 AI 网关服务。
- 后端侧：新增 AI 网关服务、管理后台 API、用量事件记录、审计日志。
- 数据层：新增 ProviderConfig、UsageEvent、AuditLog 等数据结构。
