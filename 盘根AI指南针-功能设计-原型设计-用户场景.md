# 盘根 · AI 指南针（PanGen AI Compass）

## 1. 项目定位与目标

**一句话定位**：一个面向「编程开发 / 影视创作 / 通用办公」三大领域的 AI 工具与教程聚合平台，并提供「AI 搜索推荐」与「工具组合方案生成」能力，帮助用户快速从“遇到难题”到“选工具+学教程+生成解决方案”。

**目标用户**

- 新手用户：不知道用哪个 AI 工具、缺少学习路径。
- 进阶用户：需要根据目标快速组合多工具工作流。
- 内容消费用户：通过文章/视频教程学习实践。

**边界说明（基于现有代码实现）**

- 当前数据为前端内置的 `MOCK_*`（工具/文章/主题）演示数据。
- 登录、后台、收藏等多数为“演示/占位”，非真实后端。
- AI 搜索调用 Gemini；在缺少 API Key 时回退为演示响应。

## 2. 技术栈与结构（从代码归纳）

- **前端框架**：React 19 + TypeScript
- **构建工具**：Vite
- **样式**：TailwindCSS（通过 CDN 注入）
- **Markdown 渲染**：`react-markdown`
- **图标**：`lucide-react`
- **AI 服务**：`@google/genai`（Gemini）

**核心文件/模块**

- `index.tsx`：React 入口
- `App.tsx`：应用主状态机（用 `view.type` 做页面切换），聚合全部页面/流程
- `constants.ts`：演示数据与常量（APP 名称、ICP 等）
- `types.ts`：领域/内容类型/页面状态/数据结构定义
- `components/Layout.tsx`：Header/Footer（领域切换、主题、语言、用户入口）
- `components/AISearch.tsx`：AI 搜索输入与结果区
- `components/CardComponents.tsx`：工具/文章/主题卡片
- `services/geminiService.ts`：AI 搜索的 Gemini 调用与回退逻辑

## 3. 信息架构与页面状态机（ViewState）

应用没有使用路由库，而是由 `view` 状态控制页面：

| view.type | 页面名称 | 入口/触发点 | 主要目的 |
| --- | --- | --- | --- |
| `home` | 首页 | 默认进入、返回按钮、Header 切换领域后回到首页 | 浏览工具/经验内容、AI 搜索、选择工具 |
| `tool-detail` | 工具详情 | 点击 ToolCard | 了解工具、跳转官网、看相关教程 |
| `article-read` | 文章/视频阅读页 | 点击 ArticleCard / TopicCard | 阅读内容、目录/主题联动（演示） |
| `login` | 登录页（演示） | Header 登录按钮 | 登录以保存方案/收藏（演示） |
| `solution-generate` | 方案生成页 | 浮动 Dock「生成方案」 | 输入目标，基于已选工具生成方案（演示） |
| `user-center` | 个人中心 | Header 用户菜单 | 查看资料、查看方案、导出/删除/分享 |
| `admin` | 管理后台（演示） | Header 用户菜单「后台管理」 | 占位展示 |

## 4. 功能设计（模块拆解）

### 4.1 领域（Domain）与内容类型（ContentType）

- **领域 Domain**：`creative`（影视创作）、`dev`（编程开发）、`work`（通用办公）
- **内容类型 ContentType**：
  - 工具推荐（Tools）
  - 经验心得（Experience）

**设计要点**

- Header 左侧 Logo/下拉是“领域切换器”，切换后：
  - `currentDomain` 更新
  - 自动 `navigate({ type: 'home' })` 回首页
- 首页内容根据 `currentDomain` 过滤：
  - 工具：`MOCK_TOOLS.filter(t => t.domain === currentDomain)`
  - 主题：`MOCK_TOPICS.filter(t => t.domain === currentDomain)`
  - 文章：按 `isFeatured` 分为精品/广场

### 4.2 首页（HomeView）

**组成结构**

- Hero 区：标题 + 副标题 + AI 搜索（`<AISearch />`）
- 模块切换：
  - 工具推荐（Tools）
  - 经验心得（Experience）
- 内容区：
  - Tools：网格 `ToolCard` 列表
  - Experience：`featured/plaza` 子 Tab
    - Featured：系列主题 `TopicCard` + 精选文章 `ArticleCard`
    - Plaza：大众广场文章列表

**关键交互**

- 点击 ToolCard：进入工具详情
- 点击 ArticleCard：进入阅读页
- 点击 TopicCard：进入该主题的第一篇文章（若存在）

### 4.3 工具选择与浮动 Dock（Shopping-cart 风格）

**状态**

- `selectedToolIds: Set<string>`：已选工具集合

**卡片层交互**

- ToolCard 右上角提供选择按钮（“加入方案 / 取消选择”）
- 选择按钮点击会 `stopPropagation`，避免触发进入详情页

**浮动 Dock（当 `selectedToolIds.size > 0` 显示）**

- 左侧：已选数量 + 前 4 个工具头像预览
- 右侧操作：
  - 清空：`clearSelection()`
  - 分享：弹出菜单（复制文本/分享微信/QQ，均为复制到剪贴板 + alert 提示）
  - 生成方案：进入 `solution-generate`

### 4.4 工具详情页（ToolDetailView）

**信息结构**

- 顶部大图 Banner（工具封面图）+ 评分/使用人数/领域
- 工具全描述 `fullDescription`
- 关键 CTA：
  - “访问官网”（新开窗口）
  - “收藏”（当前仅 UI）
- 相关教程：从 `MOCK_ARTICLES` 按 `relatedToolId` 过滤

### 4.5 文章/视频阅读页（ArticleReadView）

**信息结构**

- 左侧（桌面）：主题侧栏（Topic TOC）
  - 若文章属于某主题：展示主题信息 + 该主题文章列表
  - 点击不同文章：仍在阅读页内切换
- 中间：
  - 面包屑
  - 标题、作者、日期、阅读时长
  - 若为视频：展示“模拟播放器”区域
  - 正文：Markdown 渲染（`react-markdown`）
  - 评论区：2 条演示评论
- 右侧（超宽屏）：目录（演示静态项）

### 4.6 AI 搜索推荐（AISearch + geminiService）

**目的**

用户输入自然语言目标（例如“我想学剪辑短视频...”），AI 返回：

- `summary`：需求理解摘要
- `recommendation`：建议路径
- `suggestedTools`：推荐工具（按名称匹配到现有工具列表）
- `suggestedArticles`：推荐文章（按标题匹配到现有文章列表）

**交互与状态**

- 搜索提交后进入 loading（包含 800ms 演示延迟）
- 展开结果面板（可关闭并清空状态）

**回退策略**

- 若未配置 API Key：返回演示结果（不会调用外部服务）

### 4.7 方案生成（SolutionGenerateView）

**入口**

- 浮动 Dock 点击“生成方案”进入

**流程**

1. 展示已选工具标签列表
2. 用户输入目标（textarea）
3. 点击生成：
   - 显示生成中（loading）
   - 1.5s 后生成演示 `UserSolution`
   - 调用 `saveMockSolution`：保存到 `userSolutions` 并跳转到个人中心的“我的方案”

**输出内容**

- `aiAdvice` 为 Markdown 文本（在方案详情抽屉中渲染）

### 4.8 登录（LoginView，演示）

- “模拟一键登录”：将 `isLoggedIn` 置为 `true` 并回首页

### 4.9 个人中心（UserCenterView）

**入口**

- Header 头像菜单 -> 个人中心

**侧边栏 Tab**

- 个人资料（profile）
- 我的方案（solutions）
- 收藏夹（favorites，占位）

**我的方案（solutions）**

- 列表展示：标题、目标摘要、工具头像、创建日期
- 操作：
  - 导出（快捷按钮默认导出 Markdown）
  - 删除（触发确认弹窗）
- 点击条目：打开右侧抽屉查看详情

**方案详情抽屉**

- 顶部：标题、日期、分享按钮（下拉菜单）
- 内容：核心目标、涉及工具（可点击跳转工具详情）、方案建议（Markdown 渲染）
- 底部：
  - 导出菜单：Markdown / TXT / CSV
  - 删除按钮（走确认弹窗）

**删除确认弹窗**

- 强提醒 + 不可撤销说明

### 4.10 管理后台（AdminView，占位）

- 展示“仅供演示”，无真实管理能力

### 4.11 主题/护眼模式与语言

- `themeMode`：`light` / `eye-care`
  - 影响全局背景与多数卡片配色
- `language`：`zh/en/jp/ru`
  - Header 可切换（当前大多数内容未做多语言渲染，属于“预留状态”）

### 4.12 导出与下载（exportSolution / downloadFile）

- 前端 Blob 下载
- 支持导出：
  - `md`：Markdown
  - `txt`：纯文本（会移除部分 Markdown 符号）
  - `csv`：表格（包含 AI 建议字段）

## 5. 原型设计（页面结构稿 / 交互稿）

说明：这里给的是“实现对齐的低保真原型结构”，用于产品/交互对齐与后续拆分路由、抽组件。

### 5.1 全局框架

**页面骨架**

- Header（固定顶部）
  - 左：领域切换（影视创作/编程开发/通用办公）
  - 右：语言切换、登录/用户头像菜单、移动端菜单
- Main（内容随 `view.type` 切换）
- Footer（ICP/版权信息）

### 5.2 首页原型

**A. Hero 区**

- 标题：遇到难题？用 AI 解决
- 输入框：AI 搜索（placeholder：我想学剪辑短视频...）
- 提交按钮：右箭头

**B. 内容切换**

- Segmented Control：工具推荐 / 经验心得

**C. 工具推荐（默认）**

- 标题：热门工具
- 工具网格：ToolCard（含选择按钮）

**D. 经验心得**

- Tab：精品专区 / 大众广场
- 精品专区：
  - 系列主题（TopicCard）
  - 精选文章（ArticleCard）
- 大众广场：文章列表 + 空态“暂无内容，快来投稿吧！”

**E. 浮动 Dock（仅当有选择）**

- 位置：底部居中
- 内容：头像预览 + 已选数量
- 操作：清空 / 分享（弹层）/ 生成方案

### 5.3 工具详情页原型

- 返回按钮（回首页）
- Banner：大图 + 评分/使用人数/领域标签
- 工具描述正文
- CTA：访问官网 / 收藏
- 相关教程：文章卡片网格

### 5.4 阅读页原型

- 左侧（可选）：主题 TOC（仅在主题文章时出现）
- 中间：面包屑 + 标题信息 + 视频区域（可选） + Markdown 正文 + 评论区
- 右侧：目录（演示静态）

### 5.5 方案生成页原型

- 返回首页
- 已选工具标签区
- 目标输入区
- 主要按钮：生成方案（loading 状态：正在思考...）
- 顶部辅助：分享按钮（复制内容）

### 5.6 个人中心原型

- 左：用户信息卡 + Tab
- 右：
  - 我的方案列表（卡片式）
  - 个人资料表单（昵称可编辑、邮箱禁用）
  - 收藏/历史/创作者：占位

**方案详情抽屉**

- 右侧滑出
- 分享菜单（复制链接/微信/QQ/X/微博）
- 导出菜单（Markdown/TXT/CSV）
- 删除确认弹窗

## 6. 用户场景（角色、任务流、边界情况）

### 6.1 角色定义

- **游客（未登录）**：浏览、AI 搜索、选择工具、生成方案、分享文本（可用）
- **登录用户（演示）**：可进入个人中心查看/导出/删除方案
- **管理员（演示）**：可进入后台页面（占位）

### 6.2 典型场景 1：快速找工具（游客）

- 进入首页
- 选择领域（例如“影视创作”）
- 在“工具推荐”网格浏览
- 点击某工具进入详情 -> 点击“访问官网”

**成功标准**：用户能在 1-2 次点击内找到工具官网。

### 6.3 典型场景 2：AI 搜索引导学习路径（游客）

- 首页 Hero 输入目标（如“我想学剪辑短视频”）
- 点击搜索
- 查看 AI 推荐方案（summary/recommendation）
- 从推荐卡片进入工具详情或文章阅读

**边界情况**

- 无 API Key：提示演示模式，仍返回推荐内容。

### 6.4 典型场景 3：阅读系列主题内容（游客）

- 切到“经验心得”->“精品专区”
- 点击某系列主题
- 进入第一篇文章阅读
- 在左侧主题 TOC 切换下一篇

### 6.5 典型场景 4：收藏/方案式的“选型”流程（游客）

- 在工具网格为 2-4 个工具点击“加入方案”
- 浮动 Dock 出现：确认已选数量
- 点击“生成方案”进入方案生成页

### 6.6 典型场景 5：生成方案并沉淀到个人中心（登录用户/演示）

- 在方案生成页输入目标
- 点击生成
- 生成完成后自动跳到“个人中心 -> 我的方案”
- 点开方案抽屉查看 AI 建议

### 6.7 典型场景 6：导出与分享（登录用户/游客）

- 在“我的方案”列表：一键导出 Markdown
- 在抽屉底部：选择导出 Markdown/TXT/CSV
- 在分享菜单：复制链接或复制文本用于社交平台

### 6.8 典型场景 7：误操作删除的拦截（登录用户/演示）

- 用户点击删除
- 弹出不可撤销确认
- 用户可取消或确认删除

### 6.9 典型场景 8：视觉舒适与偏好（全用户）

- Header 开启护眼模式
- 全站背景与卡片色系变更为暖色系

## 7. 版本规划建议（可选，便于后续落地）

- V1：接入真实后端数据（工具/文章/主题）、收藏与登录态持久化
- V2：方案生成接入真实 AI（对工具/文章上下文做 RAG 或结构化召回）
- V3：路由化（React Router）、模块拆分（页面组件/业务 hooks）、多语言真正落地
